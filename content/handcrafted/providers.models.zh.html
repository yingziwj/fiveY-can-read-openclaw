先讲人话版

⚡ Model Provider Quickstart 这页，在教你两步就把模型电源接上

这页故意写得短，是因为它只想解决一个问题：别先研究全世界，先让 OpenClaw 确实连上一位能说话的模型老师。

顺序就两步。先拿到入场票，再告诉 OpenClaw 默认该找谁聊天。

适合刚装好 OpenClaw 的第一轮配置 两步走：认证，然后选默认模型 查看官方原文

第一站

🪪 第一步不是挑花哨型号，而是先办门票

Provider auth 放在最前面，是因为没票就进不了门。模型名再好看，也只是橱窗。

🔑 API key 路线

最常见、最直接。像把店家的钥匙收进抽屉里。

🪪 账号登录路线

有些 provider 支持登录式接入，更像刷会员卡，不一定靠你手填长期 key。

🛠 onboarding 在做什么

像带路员，先问你选哪家 provider，再帮你把门票放进正确抽屉。

🎈 一句话

先有可用认证，后面谈模型才有意义。

第二站

🧠 第二步才是挑默认模型

真正需要的是“默认先找谁回答”。官方只给 starter set，就是怕你第一天就被型号海洋淹掉。

先选一家店

比如 OpenAI、Anthropic、Google、OpenRouter、Ollama。

再选一个默认型号

像 openai/gpt-5.4 这种，本质是“店名/货号”。

先稳定再扩展

第一轮先让一个模型能答，再考虑备用模型、图片模型和快速模式。

🎈 最后记一句

Quickstart 不是百科，而是“最快请来第一位模型老师”的流程卡。

最后总结

🎈 把 Model Provider Quickstart 压成一句最好记的话

先把 provider 认证跑通，再选一个默认模型。别把顺序反过来。

跑通后，再去看你真正要用的具体 provider 页面。
