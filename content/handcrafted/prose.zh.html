先讲人话版

📖 OpenProse 不是“又一个命令”，而是一套给 AI 任务写剧本的格式

它把流程写成 `.prose` 文件，再让 OpenClaw 用插件和 `/prose` 命令去跑。你可以把它想成“把任务写成童话剧本”。

重点不是语法炫不炫，而是流程能不能重复、能不能并行、能不能交给别人照着跑。

适合想把工作流写成可复用剧本的人 先启用插件，再看 slash command，再看 `.prose` 文件 查看官方原文

第一站

🧩 先把它想成“任务剧本”

OpenProse 的厉害之处，不是写一段文本，而是把一套会反复执行的动作串成程序。

📄 `.prose` 文件

像一本任务剧本书，里面写着先做什么、后做什么、遇到岔路怎么走。

⚙️ 可并行

像让几个小演员同时上场，省时间。

🧠 可复用

同一个剧本可以拿去做研究、排障、内容流水线。

第二站

🔌 先装插件，再开命令

官方的安装步骤很短：启用插件，重启 Gateway，然后就能用 `/prose` 了。

openclaw plugins enable open-prose

像把“剧本发动机”插进主机，再把电源重新打开。

/prose run <file.prose>

像把剧本递给导演，让它按顺序开演。

/prose compile

像先排练一遍，看看剧本有没有写错。

第三站

🧵 这页真正重要的是“可控流程”

OpenProse 不是为了炫花活，而是为了让复杂工作有清楚边界：该并行就并行，该审批就审批。

🛡 审批安全

适合 code review、事故分流、内容流水线这类不能乱跳步的事情。

🪜 多代理

一个剧本能带多个小演员分头干活。

一句话

OpenProse 的价值，是把“我大概知道要做什么”变成“机器能照着跑的剧本”。
