先讲人话版

🔎 这页讲的是“给 OpenClaw 装一个搜索供应商”，不是换聊天大脑

Perplexity 在这里更像一位外聘资料员。你平时问问题时，代理需要上网查资料，就会把检索任务递给它。

所以别把它和聊天主模型 provider 混成一件事。它干的是搜资料，不是主脑推理。

适合想让代理上网搜资料的人 重点是 key 前缀决定走哪条搜索路 查看官方原文

第一步

🛠 配置命令像“把资料员招聘进办公室”

官方给了两种做法：用配置向导，或者直接把 key 填进那条很长的配置路径。

🧭

openclaw configure --section web

像走带路老师模式，让它带你去“网页搜索”那间房间填表。

✍️

openclaw config set ...apiKey ...

像绕过前台，直接在总账本上写下资料员的工牌号。

🏠

后台也得看得到 key

如果 Gateway 在后台跑，这把资料员工牌也得放到后台值班室。

🎈

这页最重要的动作

不是“手动调用 Perplexity”，而是把它配置成默认搜索供应商。

第二步

🪪 key 的前缀，就是在决定你到底走哪条搜索路线

这一点是整页最关键的信息。

pplx-...

像走 Perplexity 自家资料室，支持比较结构化的筛选和过滤。

sk-or-...

像借 OpenRouter 的门去用 Perplexity Sonar，结果更像“AI 带引用的整理答案”。

为什么官方强调前缀

因为系统会根据前缀自动挑交通工具。你给哪种工牌，它就坐哪辆车去查资料。

第三步

🧺 过滤参数在说什么？像告诉资料员“去哪找、看哪种、别翻太多”

这部分别当成参数森林，它其实很生活化。

Country / Language

像告诉资料员：“优先去哪个国家、看哪种语言的资料。”

Date range

像告诉它：“我只要最近一周，别把十年前的旧报纸也抱回来。”

Domain allow / deny

像给资料员一张白名单和黑名单，哪些书店能进、哪些别去。

Content budget

像提醒它：“别把整座图书馆都搬回来，挑关键的就行。”

最后总结

🎈 把 Perplexity Provider 这页压成一句最好记的话

这页不是换主脑，而是给代理招聘一位会上网查资料的资料员。你交哪种 key，它就走哪条搜索路线去帮你找资料。

如果你想看“代理实际怎么用这个资料员去搜”，接着看工具页；如果你只想完成招聘，这页已经够了。
