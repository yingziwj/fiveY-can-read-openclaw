先讲人话版

🛒 OpenRouter 这页，其实是在说：一个大商场门口，后面连着很多模型摊位

OpenRouter 和单一 provider 最大不同，是它更像一个总市场。你拿一把钥匙进门，里面却能试很多不同家的模型。

所以这页重点不是“怎么接一家”，而是“怎么把一个聚合入口接成你的模型总集市”。

适合想多试多比较的人 一个入口，后面连很多模型货架 查看官方原文

第一站

🔑 CLI setup 就是在给这个大商场办门票

OpenRouter 先要能进门。CLI setup 和 config snippet 讲的都是：把这张总市场门票放好。

CLI setup

像把 OpenRouter 这家商场登记进 OpenClaw 的通讯录。

Config snippet

像告诉你门票号码、默认摊位和相关规则该写在哪些格子里。

为什么很多人喜欢它

因为试模型很灵活，不用每试一家都重新接一套 provider。

🎈 一句话

OpenRouter 不是单个模型，而是一个可以逛很多模型的总商场。

第二站

📝 Notes 最重要，因为聚合入口总会带来额外规矩

好处是方便，代价是你要更清楚模型命名、计费和能力边界不完全由 OpenClaw 决定。

🏷 命名别想当然

聚合入口里，模型名字常常同时包含来源和货号，别拿别家写法硬套。

💸 价格和限制看商场规则

虽然入口统一了，但不同摊位的价格、上下文长度和能力不一定一样。

使用建议

如果你处在试模型阶段，OpenRouter 很顺；如果已确定长期用某一家，直连那家通常更稳。

🎈 最后记一句

OpenRouter 的价值不是替代所有 provider，而是把“试很多模型”变得更像逛市场。

最后总结

🎈 把 OpenRouter 这页压成一句最好记的话

它更像模型总市场：一个入口很方便，但也要接受商场级别的额外规矩。

如果你已经选定长期供应商，后面最好还是回到对应 provider 的单独页面。
