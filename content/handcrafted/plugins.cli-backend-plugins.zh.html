先讲人话版

CLI backend plugin，就是让 OpenClaw 把本地 AI 命令行当成一个模型后端

有些 AI 能力不是 HTTP API，而是已经装在电脑里的命令行工具。CLI backend plugin 像一根转接线，让 OpenClaw 可以按 provider prefix 去调用这个本地命令。

如果对方是普通网络模型 API，请写 provider plugin；如果对方管理完整代理会话，请看 agent harness。

适合本地 CLI 形态的 AI 后端核心入口：api.registerCliBackend(...)查看官方原文

怎么理解

它有三张纸：manifest、运行时登记、用户配置

manifest

像门牌，告诉 OpenClaw 这个插件存在，但它本身不执行命令。

runtime register

真正的后端行为从 registerCliBackend 开始，那里说明如何启动 CLI、传入 prompt、读回输出。

user config

用户配置像遥控器，决定 prefix、命令路径、参数和是否启用。

验证重点

先确认命令能在本机跑，再确认 OpenClaw 能找到插件入口，最后检查模型引用 prefix 是否匹配。
