先讲人话版

Channel message API，是让各聊天平台用同一套“发消息生命周期”说话

它定义 channel plugin 的 message adapter：平台怎么发送、编辑、删除、回执、线程化和处理通知，由插件说明；排队、耐久性、通用重试和共享 message tool，则由核心负责。

搭配 channel plugin 页面阅读核心子路径：openclaw/plugin-sdk/channel-message查看官方原文
Core owns

队列、耐久保存、通用重试、hooks、receipts 和共享消息工具。

Plugin owns

平台原生 send/edit/delete、target normalization、threading、quotes、通知标记和账户状态。

runtime helpers

真正做异步发送时，使用 channel-message-runtime 的 durable send helpers。

一句话

这页是在给“发消息”这件事画清职责线，避免每个平台各说各话。
