先讲人话版

🛰 `openclaw node` 像“派一个不睡觉的小工人，去别的机器上干活”

这页讲的是一个很轻的头脑风暴小帮手：它自己不负责聊天，也不负责界面，它只负责连上 Gateway，然后在别的机器上帮你跑命令。

你可以把它想成一只会跑腿的手。手不住在主机里最显眼的地方，但需要它时，它能伸到别的电脑上去做事。

适合想把命令放到远端机器执行的人 这页最重要的是记住：它连的是 Gateway WebSocket，干活时仍然要过审批 查看官方原文

第一站

🧳 为什么要有它？因为有些机器上没有大管家，但你还是想让它干活

官方一开始讲得很朴素：有些电脑不想装完整的 macOS 伴侣，但你还是想让它们帮你跑命令。node host 就是干这个的。

🖥

远端 Linux / Windows

像把一只手伸到 build server、NAS、实验机上去敲命令。

🎛

保持执行权受控

不是谁都能随便乱跑。exec approvals 和 allowlist 还在门口站着。

🪶

更轻

它是个头less 小工人，不是整套大前台。

📌

一句话先记住

node host 负责把命令送到别的机器上，不负责替你把整栋楼都搬过去。

第二站

👀 Browser proxy 像“顺手给这台远端机器配一双眼睛”

官方说得很妙：node host 还会自动带一个浏览器代理，前提是你没有把它关掉。意思就是，它不只是能跑命令，还能顺手帮你管浏览器。

🧭 自动带眼睛

如果没关，node host 会自己带上 browser proxy。像小工人出门时顺手带了一副望远镜。

browser.enabled = false

像你在出门前把那副眼镜收起来。这样这台 node 就不再负责浏览器动作。

allowProfiles

像给这双眼睛列白名单。没写进名单的浏览器房间，它就不乱进。

🚧 更严一点

如果你把代理设得更收敛，它还会挡住一些会改掉持久配置的浏览器动作。

第三站

🚀 openclaw node run 像“把这只手先叫到现场”

run 是前台开着跑。你给它 Gateway 的门牌，它就去连那边，开始工作。

▶

openclaw node run --host <gateway-host> --port 18789

像对小工人说：“现在去这台总机房门口报到。”

🔐

--tls / --tls-fingerprint

像在路上再加一层透明雨罩，还要核对这层罩是不是你认识的那一层。

🪪

--node-id / --display-name

像给这只手换工牌。名字和身份都可以换，但换了有时会清掉旧配对。

📌

一句话

run 是前台工作模式，先把手叫出来，再让它连上 Gateway。

第四站

🪙 Gateway auth 这段在说：这只手虽然轻，但也不能裸奔进门

官方特别强调 node 命令不吃 `--token` / `--password`，它要从环境和配置里找钥匙，而且找不到时会直接失败。

OPENCLAW_GATEWAY_TOKEN / OPENCLAW_GATEWAY_PASSWORD

像你先把门票放在口袋最前面。它们会优先被找到。

gateway.auth.token / gateway.auth.password

像第二层抽屉里的门票。口袋里没有时，再去抽屉找。

SecretRef unresolved

像抽屉名牌在，但里面的票没拿到。官方说这时候要直接失败，不许偷偷拿远端的票来糊弄。

gateway.mode=remote

像机器现在在远程模式里跑，某些远程客户端字段也可以参与找票。

第五站

🏠 openclaw node install 像“把这只手正式登记成长期值班员工”

run 是临时叫来，install 是把它装进系统服务，让它长期在后台待命。

openclaw node install --host ... --port 18789

像给这只手办入职手续。以后它不是临时来一下，而是正式驻扎。

--runtime node / --runtime bun

像给同一个值班员选不同的早餐车。后台用哪套运行时，可以挑。

--force

像你说“这份入职记录我就覆盖掉，重新来一版”。

🛎 管理服务

status、stop、restart、uninstall 像查勤、叫停、重新开工、解除登记。

第六站

🪪 Pairing 这段最像“门口先排队领临时工牌”

第一次连上时，node 会在 Gateway 侧产生一个待批准的设备配对请求。通过以后，这台机器才算正式入队。

1

第一次连线

像一个新工人到门口报到，先拿到待审批小条。

2

openclaw devices list

像去看门口还有哪些工牌条目正在排队。

3

openclaw devices approve <requestId>

像给那张工牌盖章，说“这台可以进”。

📦

本机记账

node 自己也会把 id、token、名字和连接信息收在 ~/.openclaw/node.json 里。

第七站

✋ Exec approvals 像“就算是帮忙干活，也得先问一句能不能动手”

node 真正执行命令时，还要过本地 exec approvals。它不是野马，手伸出去前要先点头。

~/.openclaw/exec-approvals.json

像一本允许名单。谁能动手、动到哪一步，里面都有记号。

openclaw approvals --node ...

像从 Gateway 那边把审批名单翻出来，专门给这台 node 看。

允许与允许列表

不是每个命令都能自动放。node host 会继续守住“明确、有限、可控”的那条线。

🎈 一句话

node 能跑命令，但不是无证司机。

最后总结

🎈 把 `openclaw node` 压成一句话

`openclaw node` 就是一只去别的机器上干活的轻量小工人：先连 Gateway，再按审批和白名单办事，顺手还能带一双浏览器眼睛。
