先讲人话版

Plugin architecture internals，是插件系统后台如何装箱、验票、加载的参考页

公开概念看 /plugins/architecture。这一页更像维修手册，讲启动时如何发现插件、读 manifest、挡掉不安全候选、规范配置、加载模块、调用 register，并把注册结果交给运行时。

内部机制参考关键词：load pipeline / registry / runtime hooks查看官方原文
先发现

启动时先找候选插件根目录，再读取 package 和 plugin manifest。

再验票

安全门会在运行时代码执行前挡掉危险或不合规候选。

再登记

加载 enabled native modules 后调用 register(api)，收集到 plugin registry。

兼容提示

activate 是旧别名，新插件优先使用 register。
