先讲人话版

Auth 这页是在讲“你怎么证明自己是店主”

ClawHub 网页登录用 GitHub，CLI 做发布和管理时用 ClawHub API token。网页像正门刷脸，CLI token 像给命令行的一把工作钥匙。

看懂这页，就知道什么时候用浏览器登录，什么时候手动贴 token，以及 token 放在哪里、失效了该怎么办。

适合要登录 CLI 或发布内容的人 重点是 GitHub sign-in、clh token、撤销与重登 查看官方原文

两种身份

网页认 GitHub，命令行认 ClawHub token

你先用 GitHub 在 ClawHub 网站证明“我是我”。之后 CLI 会拿到一个 clh_... 开头的 token，用它去发布、同步或查询账号信息。

GitHub sign-in

像进网站正门时出示身份证。

API token

像给命令行的小门卡，只适合放在安全配置里。

账号状态

如果账号被删除、禁用或封禁，正常登录和旧 token 都不能继续当通行证。

本机登录

clawhub login 像 CLI 开了一个临时小门等浏览器回来

默认登录会打开浏览器。你在网页完成 GitHub 登录后，浏览器会带着结果回到本机 127.0.0.1 的临时回调地址，CLI 再把 token 收好。

1

clawhub login

CLI 在本机开一个短暂的接待窗口。

2

浏览器登录

你去 ClawHub 和 GitHub 那边确认身份。

3

回到本机

登录结果回到本机回调，CLI 保存 token。

4

clawhub whoami

确认这把钥匙现在属于哪个账号。

没浏览器时

服务器和 CI 可以走 headless 路线

如果远程机器没法打开本机浏览器，就不要硬挤默认流程。可以在网页里创建 token 后传给 CLI，或者用 device flow 在另一台设备上确认。

clawhub login --token clh_...

像你已经拿到门卡，现在直接交给 CLI 保存。

clawhub login --device

像终端吐出一次性号码，你去网页上替它确认。

https://clawhub.ai/cli/device

device 登录时去这里输入那张小纸条上的码。

钥匙放哪

token 会进配置文件，不要把它晒在公开地方

ClawHub 会按系统习惯保存配置。真要换位置，可以设置 CLAWHUB_CONFIG_PATH。如果 token 被撤销或坏掉，接口会返回 401 Unauthorized，这时重新登录最稳。

macOS

~/Library/Application Support/clawhub/config.json

Linux/XDG

$XDG_CONFIG_HOME/clawhub/config.json 或 ~/.config/clawhub/config.json

Windows

%APPDATA%\clawhub\config.json
