先讲人话版

🤝 openclaw pairing 像“门口有人举着配对码敲门，你来决定放不放行”

这页讲的是 DM pairing request。翻成人话，就是某个支持配对的通道上，有人想把自己和 OpenClaw 连起来，于是先递来一张小纸条，上面写着配对码。

你用这组命令做的事很简单：先看门口排了谁，再决定把哪张小纸条盖章通过。

适合已经开了支持配对的通道，准备人工审核配对请求的人 这页最重要的是记住：它管的是“配对申请审核”，不是普通消息收发 查看官方原文

第一站

🪪 这组命令只有两个主动作：看名单，盖通过章

官方示例很短，因为这页本来就不是复杂系统。它就是门口审批台。

📋

pairing list

看看现在有哪些人在门口排队，举着哪些配对码等你看。

✅

pairing approve

从名单里挑一个码，盖章说“这位可以进”。

📨

--notify

像盖完章以后，再顺手回一句：“通过啦，你可以继续了。”

📌

一句话

list 看排队，approve 放行。

第二站

🧾 这两条示例命令，其实就是“翻排队簿”和“按号码叫号”

把命令想成办事大厅动作，会比背参数顺很多。

openclaw pairing list telegram

像对门卫说：“把 Telegram 这扇门前面排队的名单给我看看。”

openclaw pairing list --channel telegram --account work

像再补一句：“别把所有窗口都给我，只看 Telegram 里 work 这个账号负责的那一列。”

openclaw pairing approve telegram <code>

像照着小纸条上的号码叫号，然后把这一位放进来。

--notify

像办完以后再递一张回执，告诉对方审批已经过了。

第三站

🚪 `telegram` 既可以直接写在后面，也可以放进 --channel

官方专门提醒这一点，是因为很多命令行页都容易把“位置参数”和“选项参数”搞得很像两套不同玩法。其实这里不是。

1

pairing list telegram

像直接说门名，简单直接。

2

pairing list --channel telegram

像把门名放进表格栏目里，读起来更规整。

3

本质一样

都是在告诉系统：“我要看 Telegram 这扇门的配对队伍。”

📌

一句话

门名可以直接说，也可以塞进 --channel 里说。

第四站

🧷 `--account` 在说：同一扇门里，可能还有多个值班窗口

如果一条通道下面挂了不止一个账号，官方就让你用 --account 把窗口再说细一点。

一扇门，多张工牌

比如同样是 Telegram，可能有默认账号，也可能有 work 账号、support 账号。

--account work

像走到大厅里指定说：“我要找 work 这个窗口的队伍，不看别的队伍。”

不写会怎样

如果系统能很明确知道你只配了一种窗口，它就能自己猜；但窗口一多，你最好别让它猜。

审批更稳

配对这种事最怕放错门，所以多账号时把窗口说清楚，会比靠默认值更稳。

最后总结

🎈 把 openclaw pairing 压成一句话

openclaw pairing 就像门口审核台：先看谁拿着配对码在排队，再按通道和账号找到对的窗口，把该通过的人盖章放行，必要时再回一声“你已经通过啦”。
