先讲人话版

💬 所谓“聊天通道”，其实就是 OpenClaw 走进不同聊天房间的门

OpenClaw 自己更像一个会思考的大脑。但大脑就算很聪明，也得先有门，才能走进 Telegram、Discord、Slack、WhatsApp 这些地方和你说话。

这页不是在炫耀“支持的平台很多”，而是在告诉你：你已经常用哪个聊天软件，就给 OpenClaw 开哪一扇门。它不用逼你换地方聊天，而是过去找你。

这一页适合第一次选聊天入口的人 最重要的是先选对门，再研究细节 查看官方原文

第一站

🏠 先想一个画面：OpenClaw 住在屋里，通道是门

你从哪个门敲，它就从哪个门回。你在 Telegram 里找它，它就回到 Telegram；你在 Slack 里找它，它就回到 Slack。

🧠 OpenClaw 是大脑

负责理解你在说什么、决定要做什么、最后生成回答。

🚪 通道是门

负责把消息搬进来，再把回答送出去。没有通道，大脑再聪明，也没地方跟你见面。

🛎 Gateway 是门口总管

所有门基本都接到 Gateway。你可以把它想成总前台，专门负责接消息、转消息、认清是从哪扇门来的。

⭐ 这一页真正想说什么

文字聊天几乎哪里都能做，但图片、表情、群管理这些高级动作，不同平台差别很大。所以“能不能接入”和“接入后能做多少事”，是两回事。

第二站

🧭 选通道时，别一上来就全看完

这页列了很多平台，但第一次不需要全部研究。最省力的办法，是先回答一个问题：你最想让 OpenClaw 出现在哪儿？

⚡

想最快跑起来

先看 Telegram。官方也在暗示这通常是最容易上手的一扇门，因为很多时候只要一个 bot token 就能开工。

👥

想放进团队协作

先看 Discord 或 Slack。如果你的目标是频道、群组、团队流程，这两类门通常更成熟。

📱

想跟熟人日常聊天

WhatsApp、Telegram、Signal 这类会更像“把 AI 放进平时就在用的聊天工具里”。只是有些门开起来更麻烦。

🧪

只想先测试

先看 WebChat。它像一个自带的试衣间，不一定最花哨，但能先帮你确认系统有没有活过来。

第三站

📚 官方列了一长串平台，但其实可以分成几类门

如果你直接盯着那份很长的名单，会有点像看超市货架。换个分法就会顺很多。

🟢 最容易理解的门

Telegram、Discord、Slack 这种 Bot API 比较成熟。意思是平台本来就愿意给机器人留一条正规通道。

🟡 需要插件的门

Feishu、LINE、Matrix、Teams 这些很多要额外装插件。门不是没有，只是你要先自己把门框装上。

🟠 需要扫码或额外折腾的门

像 WhatsApp、某些个人账号接入方案，更像“借一个真人账号去说话”。这类门虽然很有用，但通常更麻烦，也更容易有额外限制。

🔵 官方特别提醒的一扇门

BlueBubbles 是 iMessage 的推荐方案，意思不是它完美无缺，而是“如果你要接 iMessage，优先别走旧路，先走这条更新、更完整的路”。

第四站

⚖️ 这页真正有用的地方，是帮你提前接受现实

不同通道不是“换个名字而已”。它们背后是不同平台的脾气，所以体验真的不一样。

✉️ 文字消息

这一层通常最稳定。官方基本是在告诉你：纯文本聊天，几乎哪扇门都能先用起来。

🖼 图片、媒体、表情

这一层开始分家了。有的平台天生支持得多，有的平台就比较拘束。所以别看到“支持这个通道”就自动脑补“所有功能都一样”。

👥 群组行为

群聊最容易出现“这个平台能那样，那个平台不能那样”。所以官方才专门把群组行为另外拎出来写。

🎯 读这页时该带走的心态

先把通道当“入口”，不要当“功能承诺书”。它先决定 OpenClaw 能不能进来，然后才决定它进来之后能做多少动作。

第五站

🛡 还有一个很重要的提醒：门开了，不等于谁都能进

这页后面那几条 note 看起来普通，其实都在讲安全。因为通道一旦接好，外面的人就真的有机会和你的 OpenClaw 对话了。

🔐 配对和白名单

它们像门禁系统。不是为了让人麻烦，而是为了防止陌生人突然跑来把你的机器人当公共客服用。

🧵 多个通道能一起开

这不是说系统会乱掉，而是说同一个 OpenClaw 可以同时守几扇门。谁从哪儿来，它就从哪儿回。

🧰 出问题先查什么

先查这扇门本身有没有开好，比如 token、插件、扫码、连接状态；再去查权限是不是把人挡住了；最后再看群组规则和平台限制。

🤖 模型和通道不是一回事

通道负责“从哪儿进来、从哪儿出去”；模型负责“脑子怎么想”。这两块是分开的，所以别把“聊天平台”跟“AI 模型”混在一起。

最后总结

🎈 把这一页压缩成一句最好记的话

OpenClaw 的聊天通道，不是在教你背平台名字，而是在教你给 AI 选一扇最合适的门。

如果你接下来只继续看一页，我建议先点 /channels/telegram。因为它通常最容易跑通，也最适合先把“通道到底怎么工作”看明白。
