先讲人话版

Message presentation，是把“要表达什么”先说清，再让各平台自己渲染

它是 rich outbound chat UI 的共享合同。代理、CLI、approval flows 和插件只描述消息意图；Discord、Slack、Telegram、Teams、Feishu 等通道插件负责渲染成各自原生样子。

共享消息表现合同不要把平台原生字段塞进共享工具查看官方原文
共享层

文本段落、小上下文、footer、divider 这些可移植意图放在合同里。

平台层

Slack blocks、Discord components、Telegram buttons 等由 channel renderer 输出。

为什么

同一条消息可以在不同平台长得不同，但意图应该只有一份。

一句话

核心写“信的意思”，平台负责“信封长相”。
