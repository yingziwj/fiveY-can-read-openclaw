先讲人话版

🧭 Control UI 这页，就是“坐到 OpenClaw 总控室前，知道怎么开门、怎么配对、怎么不把门弄坏”

它是浏览器里的前台，但真正重要的不是界面好不好看，而是它怎么连 Gateway、怎么做首次配对、怎么处理权限。

适合第一次打开浏览器控制台的人 先看本地打开，再看首次配对，再看 auth 和语言 查看官方原文

第一站

🚪 先把门打开，但门票要带好

本机打开最简单：浏览器直接去 Gateway 的端口。可一旦不是本机，认证就不能省。

http://127.0.0.1:18789/

像家里客厅的正门，走过去就能坐。

openclaw gateway

像先把总机房开机，控制台才会亮。

token / password

像门票和口令，出门一步都要认真带着。

第二站

🪪 第一次连进来，会先做一轮“设备配对”

新浏览器或新设备第一次来，Gateway 会先问“你是谁”，不是直接放人进柜台。

pairing required

像门卫先让你登记，不是不给进，而是先确认身份。

openclaw devices list

像翻待审批名单。

openclaw devices approve <requestId>

像在名单上盖章放行。

🌍 语言、菜单、能力面板，都是“住进来以后怎么用”

这页后面讲的，是控制台里能做什么：聊天、配置、会话、工具、语言切换。
