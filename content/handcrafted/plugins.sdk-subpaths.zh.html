先讲人话版

Plugin SDK subpaths，是插件作者拿工具时用的“分抽屉目录”

openclaw/plugin-sdk/ 下面有很多窄入口。官方建议按需要拿具体子路径，而不是把一整箱 SDK 都拖进来。

这页也帮助维护者区分公共导出、兼容导出和只给仓库内部测试用的私有入口。

SDK 子路径目录新代码不要使用 deprecated compatibility facades查看官方原文
公共子路径

插件作者可以依赖，像稳定开放的抽屉。

兼容子路径

给旧插件过渡用，新代码不该继续新增依赖。

私有本地子路径

只给 OpenClaw 仓库内部测试和实现使用，不能当公开 API。

维护检查

pnpm plugin-sdk:surface 和 boundary report 用来防止公共面悄悄变乱。
