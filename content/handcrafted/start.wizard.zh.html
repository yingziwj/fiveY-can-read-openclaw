先讲人话版

🧙 Onboarding (CLI) 像“命令行里的向导老师”

这页讲的是：如果你想在终端里把 OpenClaw 装好，带着你一步一步走的那位老师会怎么问、怎么选、怎么把配置写好。

它是推荐路线。尤其在 macOS、Linux、Windows（WSL2）上，CLI 这条路最通用，也最像工程师会走的路。

适合想在终端里完整掌握安装的人 这页最重要的是记住：它是完整流程，不是单条命令 查看官方原文

第一步

🚪 先敲门，再开始一连串的问答小步子

官方一开始就给了最短入口：openclaw onboard。你可以把它想成“老师，我准备好了，请带我走流程”。

openclaw onboard

像打开向导门。老师会坐下来，按顺序带你过每一步。

不是单一命令就结束

它是一个流。先问你想去哪，再问钥匙和房子，最后才把东西写进去。

最快先聊

如果你只想先开始聊天，官方说可以先开 Control UI，不一定要先连通道。

📌 一句话

CLI onboarding 不是一把锤子，是一位会带路的老师。

QuickStart / Advanced

🧭 这页很像“轻装版”和“全装备版”的分岔口

官方把流程分成 QuickStart 和 Advanced。一个轻，一个全，适合不同节奏的人。

🌱

QuickStart

像先带你铺好最常用的路。默认 gateway、默认 workspace、默认 auth，先跑起来再说。

🧰

Advanced

像把所有抽屉都打开。模型、workspace、Gateway、channels、daemon、skills 都能自己选。

🎛

适合谁

想快点跑通就用 QuickStart；想看清每一步、想自己控细节，就走 Advanced。

📌

一句话

先轻装起步，后面再慢慢加装备。

会发生什么

📦 官方把 onboarding 里会干的事，像菜单一样排好了

这页最有用的地方，就是把整个流程拆成几个能看懂的房间。

1. Model / Auth

先挑钥匙和脑子。API key、OAuth、setup-token，或者别的 provider，都在这一间。

2. Workspace

给小工人安家，顺便种下 bootstrap 文件。

3. Gateway

定门牌号、门锁、朝向和是不是要往外露。

4. Channels

把 Telegram、WhatsApp、Discord、Signal、BlueBubbles、iMessage 这些门都接起来。

5. Daemon

如果你愿意，它还能给你安一个后台值班员，开机后自动守门。

6. Health check

像装完房子后先开灯检查一圈，确认门还能正常开合。

7. Skills

把推荐技能和依赖也整理好，像给小工人把工具包也塞进去。

🎈 记一句

CLI onboarding 的本质就是：一次性把能用的房子和工具都搭齐。

命令翻译

🛠 这些命令在这里，不是咒语，而是“把老师带着跑”的开关

看懂它们，就能看懂整个 CLI onboarding 的节奏。

🚪

openclaw onboard

像把向导老师叫出来，开始逐项问答。

🧩

--non-interactive

像告诉老师：“这次别一题一题问我了，我要直接按脚本走。”

🧾

--json

像让老师最后把结果写成机器也能看懂的表格，但它并不等于不问你。

🧵

openclaw configure

像你之后想重新摆一遍房间时，再叫同一位老师回来。

几个特别提醒

⚠️ 官方有几句很实在的话，最好直接记住

这些提醒不是附属品，它们会直接影响你能不能顺利跑通。

Windows / WSL2

Windows 也能用，但 WSL2 更稳定。像在平地和木桥之间，官方更推荐那座更稳的桥。

Secret ref

如果你要脚本化，推荐把钥匙存成引用，而不是把明文硬塞进脚本。

Model/provider 选择

不是所有 provider 都一样，像不同店铺卖不同钥匙。选对店，后面才不会卡。

Reset / doctor

如果前面配乱了，先让医生看一眼，再决定要不要重置，不要闷头硬撞。

最后总结

🎈 把这页压成一句最短的话

Onboarding (CLI) 就是命令行里的向导老师：先选轻装版还是全装备版，再一步一步把模型、工作区、Gateway、通道、后台值班员和技能包都配好。
