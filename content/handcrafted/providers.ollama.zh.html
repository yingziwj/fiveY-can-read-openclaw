先讲人话版

🦙 Ollama 像你自己家里的本地模型机房，但要走“原生大门”，别走假 OpenAI 门

这页最重要的提醒只有一句：OpenClaw 喜欢 Ollama 的原生 API，不喜欢你去敲 /v1 那扇假装 OpenAI 的门。

门一旦敲错，工具调用就容易跑歪，甚至把工具 JSON 当普通文本往外吐。

适合本地跑开源模型的人 重点是原生 API、onboard、自动发现、本地拉模型 查看官方原文

先记最重要的一句

🚫 远程 Ollama 别写 /v1，要写原生根地址

官方专门放 warning，不是为了吓你，是因为这里真的是高频坑。

❌ 错误示范

http://host:11434/v1 像走了一扇长得像 OpenAI 的假门。

✅ 正确示范

http://host:11434 像直接走回 Ollama 自己家的正门。

为什么这么较真

因为工具调用和流式行为在原生门口更正常，假门容易让系统听错话。

第一步

🚀 Onboarding 像“带你参观机房并挑一台先开机的机器”

官方最推荐这条路，因为它不只收地址，还会顺手帮你发现模型。

🛠

openclaw onboard

像带路老师先来问：你的 Ollama 机房门牌号是什么？

🏠

默认地址

默认会先猜 http://127.0.0.1:11434，也就是这台机器本地机房。

🌤

Cloud + Local / Local

像问你要不要把云上货架和本地货架都打开，还是只用本地这排架子。

📥

自动 pull

如果你选中的模型本地还没装，向导会像仓库管理员一样帮你把它拉下来。

第二步

📦 手动路线就是“先把模型搬进机房，再告诉 OpenClaw 去用它”

如果你不走向导，最常见的动作就是先 pull 模型。

ollama pull glm-4.7-flash

像把一位老师从外面请进本地机房，确保它已经坐在自己的座位上。

ollama pull gpt-oss:20b

不是切模型的魔法语，而是“先把这台模型机器装到你家里”。

登录 ollama.com

如果你想用云上货架，也要先完成登录，不然云上仓库不会给你开门。

最后总结

🎈 把 Ollama 这页压成一句最好记的话

把 Ollama 当本地机房看就对了：走原生正门，不走 /v1 假门；先确认模型真的被搬进机房，再让 OpenClaw 去叫它上班。

如果你在比较本地机房和远程大楼，Ollama 更像自己家里的设备；NVIDIA、OpenRouter 那些更像外部大楼。
