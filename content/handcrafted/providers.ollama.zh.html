先讲人话版

🦙 Ollama 这页，其实是在说：把模型老师请到你自己家里住

和云上 provider 不一样，Ollama 更像把模型搬到本机或你自己的服务器里。它不靠远方商店发货，而是你家里先得真有这位老师。

所以这页重点不是 API key，而是本地服务有没有启动、模型有没有拉下来、地址是不是找对了。

适合本地模型和隐私优先玩法 重点是本地服务、模型发现和地址 查看官方原文

第一站

🏠 Onboarding（recommended）是在确认你家里真的住着 Ollama

推荐路径的价值不神秘，就是尽量帮你自动发现本地服务和可用模型，少走弯路。

自动发现

像 OpenClaw 先在家里巡一圈，看 Ollama 服务是不是已经在常见地址上亮着灯。

推荐原因

第一次最容易卡在“服务没开”或“模型没拉”，自动发现更容易告诉你堵在哪。

Manual setup

适合你已经知道自己在做什么，想自己手填地址、模型和规则。

🎈 一句话

Ollama 不是去云上领门票，而是先确认家里厨房真的有火。

第二站

🔎 Model discovery 和 explicit setup，在讲“自动找老师”还是“你亲自点名”

这是 Ollama 页里最实用的分叉。你可以让 OpenClaw 自动看现成模型，也可以自己写清楚要找谁。

自动发现

像去教室门口看值班表，家里已经有哪些老师，就先挑名单上的。

显式配置

像你直接点名某位老师来上课，不依赖现场探测。

Custom base URL

当 Ollama 不在默认地址时，就像老师没住在主楼，你得把新门牌号告诉 OpenClaw。

🎈 最后记一句

Ollama 的关键不是 key，而是本地服务、模型货架和地址都要对上。

最后总结

🎈 把 Ollama 这页压成一句最好记的话

Ollama 不是去外面买模型，而是先把模型搬到自己家，再告诉 OpenClaw 去哪间屋子找哪位老师。

如果只继续看一页，建议回到模型总览或你准备实际加载的那个本地模型。
