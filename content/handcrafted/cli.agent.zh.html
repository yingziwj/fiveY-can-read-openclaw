先讲人话版

🚶 openclaw agent 像“让一个 agent 现在立刻出门干一趟活”

这页讲的是单次 agent turn。你给它一句话，它去 Gateway 那边跑一圈，再把结果带回来。

它很像派出一位临时小帮手。帮手不住在这里，但这次活要谁来做、给谁回信、要不要送出去，都能在这一趟里说清楚。

适合要直接触发一次 agent turn 的人 这页最重要的是记住：它是“跑一趟”，不是“长期在线的桌子” 查看官方原文

第一站

🚀 这趟活最常见的几种走法

官方例子不多，但都很像现实里会敲的命令。

📤

--to + --message

像给某个号码发一句话，请它帮你回一轮。

🧠

--agent ops

像指名让 ops 那位工人来干这次活。

🧾

--session-id 1234

像直接告诉它“这次去第 1234 个抽屉里接着干”。

🎯

--thinking medium

像这趟路让它想得中等多一点，不是最省力，也不是最慢。

第二站

🪄 `deliver` 像“这次真的把回信递出去”

很多人第一次看这页会以为只是“让 agent 想一想”。其实它还能把结果送回指定通道。

--deliver

像把这次跑出来的话，不只留在自己脑子里，而是真的寄到外面去。

--reply-channel slack

像告诉邮差：回信走 Slack 这条路。

--reply-to "#reports"

像再补一句：投到 reports 这个小房间里。

📌 一句话

deliver 打开以后，这次活才算真的送到了门外。

第三站

🧠 `--local` 像“这次不走外派，直接在本地小屋里跑”

官方这页的核心还是 Gateway，但它也提醒你有 embedded/local 的做法。也就是说，这趟活可以在本地模式里执行。

🏠

local

像小帮手在你眼前的这间屋子里直接干活。

🌉

gateway 路线

像大多数时候还是要先过 Gateway 这座桥。

🎛

`--agent` 直接指名

像这次不是随机派人，而是直接叫某个工位的人出门。

📌

一句话

agent 是一次性出门干活，不是长驻桌子。

第四站

🪪 `models.json` 重新生成那段，像“把这趟活用到的牌子换成不泄密的小标签”

官方特别提醒：当这条命令触发模型配置重新生成时，它不会把密钥明文写出来，而是写可追踪的标记。

🔐 非明文

像把门票收进小标签里，不把原始钥匙摊在桌面上。

🧾 marker

像留一张说明卡：这把钥匙是从哪里来的，但不把钥匙本身写出来。

source-authoritative

意思很简单：它信的是源配置，不是临时跑出来的明文结果。

🎈 一句话

这页很谨慎：能记来源，但不把机密直接摊开。

第五站

🎯 这条命令最适合干什么？就是“临时派一位 agent 出门回个话”

你可以用它做状态回复、日志总结、消息转发、或者指定某个 session 接着干。

📬

回消息

像临时派人去替你回一句话。

📝

做总结

像让它把一堆内容揉成一段能看懂的话。

🧵

接着一段 session

像从同一个抽屉里继续干，不从头重来。

🎈

一句话

agent 是临时出勤，不是长期驻场。

最后总结

🎈 把 openclaw agent 压成一句话

openclaw agent 就是临时派一个 agent 出门跑一趟：你告诉它去哪、想什么、回到哪儿，它就替你把这次活跑完。
