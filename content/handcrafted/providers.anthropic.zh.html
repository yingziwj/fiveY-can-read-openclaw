先讲人话版

🪶 Anthropic 这页，其实是在讲 Claude 怎么领进门、怎么跑得稳、怎么省一点

这页不只是在讲 Anthropic API key。它还在讲 Claude 默认怎么想、怎样切快一点的跑法，以及提示缓存这种“别重复花钱”的省力法。

如果你想把 OpenClaw 长期接到 Claude，这页是很典型的一类 provider 页面：既讲接入，也讲跑法。

适合准备长期用 Claude 的人 重点不是只有 key，还有思考深度和缓存 查看官方原文

第一站

🔑 Option A 说白了就是：把 Anthropic 的钥匙放进去

最普通、最稳的路线还是 API key。OpenClaw 只是帮你把钥匙收好，然后默认去 Anthropic 家找 Claude。

openclaw onboard --auth-choice anthropic-api-key

像直接告诉 onboarding：“这次我要走 Anthropic 钥匙路线。”

CLI setup

就是把钥匙和默认 provider 接好，别让 OpenClaw 以后每次都猜你要去哪家店。

Claude CLI config snippet

本质是示范相关字段该放在哪些抽屉，不是让你逐字死抄。

🎈 一句话

先把 Claude 的门票放对地方，后面再讨论想得深还是跑得快。

第二站

🧠 Thinking defaults 和 prompt caching，都在讲“同一个 Claude 怎么跑”

这不是换模型，而是在同一位模型老师身上切不同节奏。

深一点

像让老师多打草稿，答案可能更稳，但速度和成本也会往上走。

快一点

Fast mode 更像让同一位 Claude 切到低延迟跑法。

🧾 Prompt caching

像把常用课本先摊在桌上，后面别每轮都整本重搬。

🎈 最后记一句

Anthropic 这页最关键的不是“能不能接上”，而是接上以后怎样想深一点、快一点、省一点。

最后总结

🎈 把 Anthropic 这页压成一句最好记的话

Claude 这页的重点不止是认证，还包括默认思考深度、快速模式和缓存。

如果你要长期用 Claude，这几个按钮都值得单独试一轮。
