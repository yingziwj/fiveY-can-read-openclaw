先讲人话版

📱 Xiaomi MiMo 这页，就是“拿小米 MiMo 的 API key 去敲 OpenAI 兼容门”

整体路线很直：去 MiMo 控制台拿 key，然后告诉 OpenClaw 默认请哪位 MiMo 老师上场。

真正值得看懂的，是三位老师的分工：快聊、深想、能看图。

适合想接 MiMo 模型的人 重点是 XIAOMI_API_KEY 和三款 mimo-v2 模型 查看官方原文

第一步

🔑 先去 MiMo 控制台领钥匙

这页最前面的动作没有弯弯绕，就是拿 key。

🪪

XIAOMI_API_KEY

像 MiMo 模型大厅的入场券。

🛠

openclaw onboard --auth-choice xiaomi-api-key

像让向导直接来收这张门票。

⚡

非交互式模式

长命令只是“别问我问题，我把门票直接贴给你”的脚本版。

🏠

后台也要看得到

如果 Gateway 是后台值班员，这张门票也得放到后台抽屉里。

第二步

🧠 三款模型，像三位分工不同的老师

官方给你的模型概览，翻成人话其实很好记。

mimo-v2-flash

像快聊老师，速度优先，适合当默认文本模型。

mimo-v2-pro

像戴思考帽的老师，擅长更重的推理任务，而且桌子也更长。

mimo-v2-omni

像会看图的老师，除了文字，还能处理图片输入。

🎈 先别被 token 数字吓到

看那些超长上下文数字时，把它想成“桌子有多长”就够了。

第三步

🏠 配置片段其实在说三件事：门牌号、说话口音、默认老师

你看到那段配置，不用一行行背，只要认出这三件事。

https://api.xiaomimimo.com/v1

像 MiMo 这栋楼的门牌号。

api: openai-completions

像进门后统一讲 OpenAI 这套话。

xiaomi/mimo-v2-flash

像告诉 OpenClaw：“默认先叫快聊老师来。”

reasoning / image

这些不是玄学标签，而是在告诉你这位老师会不会多想、能不能看图。

最后总结

🎈 把 Xiaomi MiMo 这页压成一句最好记的话

先领 MiMo 门票，再决定默认请快聊老师、深想老师，还是会看图的老师。整体走的是一扇标准 OpenAI 兼容门。

如果你现在在比“快聊”和“推理”两档模型，这页是个很清楚的样板：先看角色分工，再看参数。
