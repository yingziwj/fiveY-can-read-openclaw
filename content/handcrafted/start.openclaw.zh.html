先讲人话版

📱 这页其实是在教你：给 OpenClaw 配一个专属“助理号码”

这里说的不是普通安装，而是更像“给自己养一个长期在线的小助理”。它有自己的 WhatsApp 号码，有自己的工作空间，会一直待命，等你发消息给它。

所以这页最重要的，不是炫酷配置，而是先把边界划清楚：这个助理应该用自己的号码，不该偷听你的私人号码。

适合想做“个人 AI 助理”场景的人 这页的关键词是专属号码、安全边界、长期在线 查看官方原文

第一站

🛡 这页最该先读的，不是功能，是“别乱接”

官方一开头就在踩刹车，这不是客气，而是在救你。因为你一旦把助理接进真实聊天世界，它就真的有机会读消息、回消息、碰工具、动文件。

🚫 别用你自己的私人 WhatsApp

如果你把个人号直接接进 OpenClaw，别人发给你的每一条消息，都可能变成 AI 输入。这几乎从来都不是你真正想要的结果。

📵 给它一个专属号码

最好的做法是单独给助理准备一个号码。这样像给它发一部工作手机，而不是让它偷看你自己的手机。

🔐 allowFrom 那句到底在说什么

它的意思是：不是谁都能给这个助理派活。你得先把允许的人写进白名单里，像门卫手里的那张名单。

💓 heartbeat 那句为什么这么凶

因为 heartbeat 会让助理主动跑任务。你还没完全信任它之前，先关掉，就像新招来的员工先别让他夜里自己乱巡逻。

第二站

📞 官方最推荐的，是“双手机模式”

这页那张图其实非常重要，它不是装饰，而是在讲最不容易出事的架构。

👤 你的手机

还是你自己的生活手机。你用它去给“助理号”发消息，像老板给员工派任务。

🤖 助理手机

它只负责当助理的 WhatsApp 号码。别人联系这个号码，就是在联系你的 AI 助理，不是在联系你本人。

💻 你的电脑

OpenClaw 跑在这儿。助理手机通过扫码把 WhatsApp Web 连到这里，电脑才是那个真正接住消息、调用模型、生成回答的大脑。

🎈 最后记一句

你和助理，不该共用同一个号码。最好是你有你的手机，它有它的工作机。

第三站

⚡ 快速开始，其实就是三步半

这页虽然写了很多内容，但第一次真的要动手时，你只要先跑通最小闭环。

🎬 把这几步想成一个小剧情

先让助理号和电脑牵上线，再把总控台开起来，然后在配置里写清楚“谁能跟它说话”，最后从你的白名单号码发一条消息过去试试。

1

先扫码登录

让助理手机通过 WhatsApp Web 认出这台电脑。

2

再开 Gateway

总控台没开，前面扫码再成功也只是连上了门，后面没人值班。

3

写最小配置

尤其是把 allowFrom 写进去，这一步决定谁能真的跟它讲话。

4

发第一条消息

当白名单里的号码发过去，它回了你，这才叫真的活了。

第四站

🛠 这些命令到底在现场做什么

这一页的命令不该只看字面。你一旦把画面想清楚，就不容易乱接。

openclaw channels login

像把电脑摆到助理手机面前，说：“来，认一下脸。”扫完码以后，这台电脑才算拿到了替助理号收发 WhatsApp 消息的资格。

openclaw gateway --port 18789

像把总控台开机，并告诉它值班窗口开在 18789 这个门牌号。总控台不开，消息即使进来，也没人接。

allowFrom: ["+1555..."]

像在门卫名单上写下“只有这个号码能进来派活”。这不是装饰字段，是安全底线。

openclaw dashboard

像重新打开控制台面板。之前 onboarding 自动帮你开过一次，后面想再进，就按这个按钮。

第五站

🧠 工作空间就是助理的大脑抽屉

官方中间这一大段在讲 workspace。别把它看成普通文件夹，它更像助理的脑袋里那几层抽屉。

📁 为什么重要

这里放着它的身份、性格、工具规则、长期记忆。换句话说，这个文件夹决定它像谁、会怎么做事。

📝 那些 md 文件像什么

SOUL.md 像性格卡，TOOLS.md 像工具规则表，IDENTITY.md 像身份卡，HEARTBEAT.md 像定时巡逻手册。

openclaw setup

像一键帮你把这些抽屉先搭好，让助理不至于脑袋空空地上岗。

📦 为什么官方建议 git

因为这套“大脑抽屉”以后会越改越重要。用 git 像给它的大脑做版本备份，哪天改坏了也能找回来。

第六站

🎛 那份长配置，不是在堆参数，而是在调助理的脾气

看到大段 JSON5 很容易头大，但你翻成人话以后，会发现它只是在决定几件事情。

🧠

用哪个模型

model 决定这颗脑袋是谁。

🤔

想多想少

thinkingDefault 决定它默认是快答，还是先多想一会儿。

👥

群里怎么开口

requireMention: true 像让它学会礼貌，群里被点名才说话。

🔄

会话怎么重置

/new、/reset 这些像“重新开一张新纸”。

🎈 一句话收住

这份配置不是为了显得复杂，而是在告诉助理：你是谁、你能跟谁说话、你在群里要不要插嘴、你多久忘一次旧上下文。

第七站

💓 heartbeat、media、状态命令，这些都是“长期上岗后”的事

后半页有很多高级内容，不是不重要，而是更像正式上岗后的管理手册。

💓 Heartbeat

像让助理定时自己巡视。很强，但也容易烧 token，所以前期先关掉是对的。

🖼 Media

像让它不只会说话，还会看图、听音频、发图片。MEDIA: 那种写法，本质就是告诉系统“这一行不是普通文本，是要顺手带出去的附件”。

🩺 状态命令

openclaw status、openclaw health --json 这些更像体检工具。等助理长期跑起来以后，你会靠它们看它有没有生病。

🎈 一句话收住

先把助理招进来，再教它高级动作；别还没入职就让它夜里自己值班。

最后总结

🎈 把这页压缩成一句最好记的话

这页真正讲的不是“怎么把 WhatsApp 接进来”，而是“怎么给 OpenClaw 配一个有自己号码、自己大脑、自己边界的长期助理”。
