先讲人话版

🪵 Logging 这页，就是“OpenClaw 的日记本放哪儿、怎么翻、怎么只看重点”

它讲两种日记：文件里的 JSONL 日记和终端/控制台里流出来的现场记录。你要的不是一堆术语，而是知道去哪儿看、怎么跟踪。

适合排障、监控、接告警的人 先看文件位置，再看 `openclaw logs --follow`，最后看 UI 和过滤 查看官方原文

第一站

📍 日记本放在哪儿

默认日志会写到 `/tmp/openclaw/openclaw-YYYY-MM-DD.log`，像每天换一页日记纸。

默认路径

像系统自动在临时文件夹里开了一本当天专页。

自定义路径

在 `~/.openclaw/openclaw.json` 里把 `logging.file` 指到别处，就像把日记本搬进自己的抽屉。

日期用 host 本地时区

日记页怎么切页，按主机自己的本地钟走。

第二站

👀 最推荐的看法，是实时追着看

官方把 `openclaw logs --follow` 放在最前面，因为它最像“边写边看”。

openclaw logs --follow

像搬一把小椅子坐到机房窗边，日志一行行往外冒。

TTY / 非 TTY

终端环境下会更漂亮，管道里则自动变成朴素文本。

--json

像把日记改成标准表格，给机器继续处理。

第三站

🧰 控制台和日志过滤，是“找重点”不是“全翻完”

这页后半段就是在教你别被日志海淹没。

Control UI 日志页

像网页里的观察窗，直接看同一份日记。

channel-only logs

像只翻某一类门口的记录本，比如只看 Telegram 或 WhatsApp。

`openclaw doctor`

如果日志拉不出来，先让医生帮你看基础环境是不是坏了。
