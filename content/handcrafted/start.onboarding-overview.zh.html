🗺 Onboarding Overview 像“先把整条新手路铺平”

这页不是讲某一个按钮怎么按，而是在帮你挑路：你要走命令行这条路，还是走 macOS app 这条路。

你可以把它当成分岔路口的路牌。先选路，再往下走，后面的细节自然就顺了。

适合第一次决定“我该怎么开始”的人 这页最重要的是记住：两条路都能通向同一个结果 查看官方原文

第一眼

🚦 这里不是“谁更高级”，而是“你今天要走哪条路”

官方说得很直白：CLI onboarding 和 macOS app onboarding 都能把 auth、Gateway、channels 配好，只是入口不一样。

⌨️

CLI onboarding

像走楼梯。适合服务器、头less、想把每一步都看清的人。

🪟

macOS app onboarding

像坐电梯。更直观，适合桌面 Mac 用户。

🏁

最终目的相同

不管从哪条路进去，最后都是把 Gateway、auth、workspace 和 channels 配好。

🎈

一句话

先选路，再走路。别在门口纠结太久。

两条路

🛣 官方把路分得很清楚：一条更像命令行，一条更像图形界面

这不是二选一的考试，而是按你的习惯选路。

CLI onboarding

适合 macOS、Linux、Windows（WSL2 更推荐）。像带着地图自己走，控制力更强。

macOS app onboarding

只适合 macOS。像有个带路老师在旁边一边问一边点按钮。

CLI 更适合什么

服务器、远程机、脚本化、喜欢清楚看到每一步的人。

app 更适合什么

桌面 Mac、想要图形界面、想少记几个命令的人。

共同任务

这页真正有用的地方，是把“会被配到什么”讲得很清楚。

🔑

模型和认证

像先决定你要拿哪把钥匙进门，再决定门卡怎么放。

🏠

工作区

像把小工人的家先安稳下来。

🚪

Gateway

像总机房的门牌、门锁、开关和朝向。

📮

Channels

像把 Telegram、WhatsApp、Discord 这些消息门都接好。

怎么选

🎯 官方其实给了你一个很朴素的答案

大多数人应该先走 CLI onboarding。为什么？因为它到处都能跑，而且你能更清楚地看到每一步。

CLI onboarding

像一条最通用的小路。屋里、服务器上、远程机上都能走。

macOS app onboarding

像一条更直观的小径。你能边看界面边点选，心理更轻松。

最常见的起点

如果你不确定，就先走 CLI。这是官方自己推荐的默认答案。

📌 一句话

先能通，再求漂亮。先把路走通，比站在门口比较路牌更重要。

细节提醒

⚠️ 这页还偷偷告诉你几个很实用的小事实

这些小事实不花哨，但能帮你少走很多弯路。

🧪

--json 不是非交互

官方特别提醒：想让脚本自动跑，要用 --non-interactive，不是只加 --json。

🔌

自定义 provider 也可以共存

像你在同一栋楼里放了好几个门牌，不同 endpoint 可以并排住着。

🧭

CLI 最适合自动化

脚本、服务、远程机、无头环境，这些都更适合从 CLI 起步。

🪟

macOS app 更像现场带路

适合你想边看边点，不想一口气记住太多命令的时候。

最后总结

🎈 把这页压成一句最短的话

Onboarding Overview 就是新手路口的路牌：CLI 和 macOS app 都能把你带进门，先选你顺手的那条，再去配同样的核心东西。
