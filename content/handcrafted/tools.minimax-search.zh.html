先讲人话版

MiniMax Search 像给 Web Search 换上一位 MiniMax 外勤同学

OpenClaw 的 web_search 可以接不同 provider。MiniMax Search 这页讲的是：怎样用 MiniMax Token Plan search API 带回标题、链接、摘要和相关问题。

MiniMax 网页搜索提供商 重点是 key、region、环境变量、参数边界 查看官方原文

第一站

先拿工牌：Token Plan key

MiniMax Search 需要凭据。你可以从 MiniMax Platform 创建或复制 Token Plan key；如果是 OAuth 设置，也可以复用 MINIMAX_OAUTH_TOKEN。

放在哪里

可以放进 Gateway 环境变量，比如 MINIMAX_CODE_PLAN_KEY，也可以通过 openclaw configure --section web 配置。

Gateway 安装

如果是 Gateway 部署，环境变量通常放在 ~/.openclaw/.env。

其他可识别变量

MINIMAX_CODING_API_KEY、MINIMAX_OAUTH_TOKEN、MINIMAX_API_KEY 也可作为替代来源。

一句话

没有 MiniMax 工牌，OpenClaw 就不能派这位外勤去查网页。

第二站

Region 像选择服务窗口：Global 还是 CN

MiniMax Search 有 global 和 CN 两个 endpoint。OpenClaw 会根据配置和凭据线索来决定该去哪一个窗口。

Global

https://api.minimax.io/v1/coding_plan/search

CN

https://api.minimaxi.com/v1/coding_plan/search

未设置 region

如果没写明 region，OpenClaw 会按自己的顺序推断。读配置时先看有没有显式设置。

为什么重要

选错窗口可能表现为认证失败、访问失败或结果不符合预期。

第三站

它是 provider 页面，不是搜索技巧大全

这页重点是让 MiniMax 接入 web_search。provider-specific filters 目前不支持，所以别期待这里有很多 MiniMax 专属过滤器。

返回什么

结构化搜索结果，包括标题、URL、snippet 和 related queries。

和总览页关系

所有 provider 怎么选、web search 和 web fetch 怎么分，回到 /tools/web 看总览更合适。

和 MiniMax provider 页关系

模型、图像、语音和认证整体设置，看 /providers/minimax。

一句话

这页只解决“把 MiniMax 当成网页搜索外勤接进 OpenClaw”。

最后总结

把 MiniMax Search 压成一句话

准备好 MiniMax Token Plan key，选好 region，把它接到 web_search，OpenClaw 就能让 MiniMax 带回结构化网页线索。
