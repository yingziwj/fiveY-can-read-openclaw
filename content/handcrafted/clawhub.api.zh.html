先讲人话版

API v1 像 ClawHub 货架的“公共查询窗口”

如果网页是给人逛的店面，API 就是给程序员和小机器人用的窗口。它能查 skill、搜索、下载，也能让登录后的发布者管理自己的东西。

这页最重要的不是背完每条接口，而是分清三件事：哪些能公开看，哪些要带钥匙，什么时候要慢一点别把窗口敲爆。

适合想接 ClawHub 数据的人 重点是 public read、Bearer token、rate limit 查看官方原文

入口地址

所有新版窗口都从 /api/v1 进去

官方把根地址写得很清楚：默认站点是 https://clawhub.ai，新版 API 都放在 /api/v1/... 下面。

https://clawhub.ai

像整座仓库的大门。

/api/v1/openapi.json

像给程序看的接口地图，告诉它每个窗口怎么说话。

旧接口还在

/api/... 和 /api/cli/... 是老门牌，主要为了旧工具还能回家。

公开查询

不用登录也能看货架，但要按规矩看

公开读接口适合做第三方目录、搜索页或小型查询工具。可以看公开信息，但不能假装自己就是 ClawHub，也不能绕过隐藏和审核边界。

S

GET /api/v1/search

像在货架上输入关键词找 skill。

L

GET /api/v1/skills

像翻一页一页的公开商品清单，可以按更新时间、下载量或趋势排序。

D

GET /api/v1/skills/{slug}

像拿起某一盒 skill，看名字、版本、作者和安全状态。

F

GET /api/v1/skills/{slug}/file

像只翻看盒子里某一张说明纸，有大小限制，不是无限复印机。

需要钥匙

发布、删除、改名、转让，都要带 Bearer clh_...

这些动作会改变货架，所以 ClawHub 必须先确认“你是谁”。没有 token、token 过期，或者账号状态不正常，系统就会拒绝。

Authorization: Bearer clh_...

像把店主钥匙递给柜台看。它不该写进公开网页或仓库。

POST /api/v1/skills

发布新版本，像把整理好的新盒子放上架。

DELETE /api/v1/skills/{slug}

软删除 skill，像临时撤下货，不一定立刻把名字让出去。

GET /api/v1/whoami

像问柜台：“我现在这把钥匙到底是谁的？”

别敲太急

rate limit 是在保护大家共用的窗口

公开读取、登录读取、写入和下载都有自己的节奏限制。遇到 429 时，程序应该等一等，而不是更用力地重试。

匿名读取：600/min

没登录时按 IP 算，像一群人共用同一张排队号。

登录读取：2400/min

带有效 token 时按用户算，额度更清楚。

写入：45/min 或 180/min

改货架的动作更慢，因为出错影响更大。

Retry-After

像柜台递的小纸条：“请等这么多秒再来。”客户端最好照做。
