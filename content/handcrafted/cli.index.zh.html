先讲人话版

🧰 CLI Reference 像一整盒工具抽屉地图

这页不是教你立刻做某一件事，而是在告诉你：`openclaw` 这把瑞士军刀，刀片都藏在哪些抽屉里。

如果你把 OpenClaw 想成一栋楼，这页就像楼层索引。想装机器、查健康、看日志、管会话、调浏览器、接 Gateway，都能从这里找到门。

适合刚打开命令行，还不知道该先敲哪条的人 这页最重要的是先分清：安装类、查看类、控制类、扩展类，是四种不同动作 查看官方原文

第一站

🪄 先把 openclaw 想成“总工具箱入口”

你敲的不是一串散命令，而是在打开一个总工具箱。后面跟什么词，就像伸手去摸不同抽屉。

openclaw setup / onboard

像先把新家布置好，让 OpenClaw 能住进来、亮起来。

openclaw status / health / logs

像去看体检单、看仪表盘、看监控屏。

openclaw gateway / sessions / agents

像去机房和值班室管理后台怎么跑、桌子怎么分、抽屉怎么开。

openclaw browser / plugins / skills

像给它加手、加眼睛、加新道具。

第二站

🧭 最容易迷路的地方，是把“先搭起来”和“已经在运行”混成一件事

这页列了很多命令，但真正好记的方法是按场景分。

🏗

搭起来

setup、onboard、configure 像装修和领钥匙。

🩺

看状态

status、health、doctor、logs 像查体温、问病情、翻监控。

🎛

控后台

gateway、sessions、agents、system 像总控室按钮。

🧩

加能力

plugins、skills、browser、mcp 像往工具箱里继续塞新零件。

第三站

🎨 Global flags 其实是在讲：这次要不要开测试小屋、换一套家当、把彩灯关掉

这些不是业务命令，而是整把工具箱的总开关。

--dev

像临时进测试小屋。状态目录和默认端口都会偏到一套更安全的试玩区域。

--profile <name>

像拿另一把钥匙进另一套房。你还是你，但配置、状态、家当是另一份。

--no-color

像把工具箱上的彩灯关掉，只留下纯文字，方便机器抄走或人眼睛休息一下。

--update / --version

前者像顺手查有没有新版零件，后者像看看你手上这把工具箱是哪一代。

第四站

🌳 Command tree 不是让你背，而是让你知道“抽屉里还有小抽屉”

比如 openclaw gateway 下面还有 status、probe、restart。这就像大抽屉里继续分小格子。

backup create / backup verify

像先打包，再验包是不是没坏。

gateway status / gateway probe

像一个是问值班室“你还在吗”，另一个是拿着手电到处照一圈。

plugins install / plugins inspect

像先把新玩具搬进来，再拆开看看它到底有几只手、几条腿。

🎈 一句话

看到空格继续变长时，不要慌，那只是说明你还在往更细的小抽屉里走。

第五站

💡 真正的使用顺序，通常不是乱敲，而是这条小路线

你第一次上手，大多数时候不会从树顶一路往下背，而是按问题走。

1

先搭

setup 或 onboard

2

再看活没活

status、gateway status、health

3

有问题就翻监控

logs、doctor

4

再加能力

plugins、skills、browser、mcp

最后总结

🎈 把 CLI Reference 这页压成一句话

CLI Reference 就是 OpenClaw 命令行的抽屉地图：先知道哪类命令负责哪类动作，你以后看到长命令时就不会觉得它像一团绳子。

如果你下一页只想继续看一页，我建议看 /cli/gateway。因为大多数“先让它跑起来”的第一站，最后都会落到 Gateway 这间总控室。
