先讲人话版

🧠 Skills 这页，其实是在讲“Mac app 自己不解析技能，它只是通过 Gateway 把技能信息拿来展示和安装”

也就是说，技能的脑子不在 Mac app 本地，而在 Gateway 那一侧。

Mac 上的 Skills 入口重点是 skills.status / skills.install 都走 Gateway查看官方原文

第一站

📦 Mac app 更像技能商店橱窗，不是技能解析器本体

它负责展示“有什么技能、缺什么依赖、点安装时怎么转发给 Gateway”。

skills.status

像从 Gateway 拉一份技能货架清单回来，顺便看哪些还缺前置条件。

metadata.openclaw.requires

这是技能门口的小条件牌，告诉系统这项技能还缺什么。

skills.install

点安装时，真正干活的还是 Gateway 主机，不是 Mac app 自己悄悄装。

brew / npm / uv / go

这些像不同的配送路线，具体走哪条由 skill metadata 决定。
