先讲人话版

HTTP API 是 ClawHub 仓库的“完整服务柜台手册”

这页很长，因为它不只讲怎么搜索，还讲版本、文件、下载、发布、包、审核、举报、转让、npm 镜像和旧接口。它像后台柜台的厚手册。

低龄读者不需要一次背完。先记住：查东西走公开窗口，改东西带 token，下载和搜索要遵守速度，安全和审核窗口会保护大家。

适合写自动化或第三方目录的人 重点是 v1 路径、分页、下载、发布、审核边界 查看官方原文

先看地图

根地址、OpenAPI、旧门牌，先别混

默认根地址是 https://clawhub.ai，新版路径都在 /api/v1/...。旧的 /api/... 和 /api/cli/... 还留着，是为了老 CLI 不会突然找不到家。

/api/v1/openapi.json

像机器可读的柜台说明书。

/.well-known/clawhub.json

像门口公告牌，CLI 可以从这里发现 registry 和 auth 地址。

canonical URL

展示第三方目录时，最好链接回 https://clawhub.ai/<owner>/<slug>。

搜索和列表

找 skill 不是只看谁最火，还要看谁最匹配

search 会综合文字匹配、slug/name 命中和下载量先验。下载量能帮忙打破平局，但不会把不相关的东西硬推到最前面。

Q

q

搜索词。发布者应该把用户真会搜的词写进名字、简介和标签。

F

nonSuspiciousOnly

像打开“先别显示可疑物品”的过滤开关。

P

limit + cursor

像分页翻清单。短页不一定代表结束，要看 nextCursor。

T

trending

趋势排序看最近 7 天的安装信号。

详情、版本、文件

一个包不只是名字，还要看版本和里面的说明纸

详情接口会给出 skill 或 package 的基本资料、版本、作者、平台元数据和审核状态。文件接口只给原始文本，并且有大小限制。

GET /api/v1/skills/{slug}

看某个 skill 的首页资料和最新版本。

GET /api/v1/skills/{slug}/versions

像翻历史版本本子，知道它一路怎么变。

GET /api/v1/skills/{slug}/file

拿某个文本文件，默认看最新版本，文件不能太大。

GET /api/v1/skills/{slug}/scan

看安全扫描和审核状态，别把历史版本和当前状态混着判断。

下载和镜像

下载窗口会给文件，也会给完整性线索

skill 下载通常走 /api/v1/download。plugin/package 既有旧 ZIP 下载，也有 ClawPack npm tarball 镜像。恶意、隔离或私有内容会被拦住。

GET /api/v1/download

下载某个 skill 版本，version 和 tag 可以指定版本。

/packages/{name}/download

给旧客户端用的确定性 ZIP 包。

/api/npm/{package}

像 npm 能读的包目录，只列出 ClawPack-backed 版本。

发布与管理

写入接口都是“改货架”，必须拿有效 token

发布 skill 或 plugin 时，官方更推荐 multipart/form-data：一边放 payload JSON，一边放文件。改名、合并、转让和删除也都需要确认权限。

P

POST /api/v1/skills

发布 skill 新版本，像把新盒子正式上架。

K

POST /api/v1/packages

发布 code-plugin 或 bundle-plugin，还会检查 manifest、兼容性和构建元数据。

R

rename / merge

像换门牌或把旧货架并到新货架，旧链接会尽量指向新位置。

T

transfer

把 skill 交给别人，需要对方接受，也会有过期时间。

安全和审核

举报、申诉、隔离，是仓库里的安全巡检流程

ClawHub 允许社区发布，但不等于任何东西都能无条件展示和下载。报告、申诉、triage、quarantine、revoke 这些词，本质上是在处理“这盒东西是否安全、是否该继续给大家拿”。

POST /skills/{slug}/report

用户报告可疑 skill，像按下“请管理员看一眼”。

POST /skills/{slug}/appeal

发布者申诉，说明为什么这个结果可能误会了。

moderator/admin endpoints

只给管理员或审核员用，普通客户端不要当成公共窗口。
