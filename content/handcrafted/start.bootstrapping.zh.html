先讲人话版

🍼 Bootstrapping 像“第一次入住时的开箱小仪式”

这页讲的是第一次跑 agent 时，OpenClaw 怎么把小工人的房间先摆好，再问它几个最基础的问题。

你可以把它想成新家开箱。先放床、放桌子、贴名字，再让小工人正式坐下来上班。

适合想弄懂“第一次开工时，OpenClaw 在忙什么”的人 这页最重要的是记住：bootstrapping 只跑一次 查看官方原文

第一幕

🎬 第一次开工时，桌上会先多出几张小纸条

官方说得很清楚：bootstrapping 是 first-run ritual。意思不是每次都来一遍，而是第一次来时做一次开场准备。

📦

种好基础文件

会先放进 AGENTS.md、BOOTSTRAP.md、IDENTITY.md、USER.md。

❓

一题一题问

它不会一口气塞你一整本问卷，而是一道一道慢慢问，像耐心的小老师。

✍️

写进身份卡

问完后，它会把名字、习惯和偏好写进 IDENTITY.md、USER.md、SOUL.md。

🧹

只跑一次

BOOTSTRAP.md 做完仪式后就会被拿掉。它像一次性的开荒封条。

发生地点

🏠 这个仪式只在 gateway host 上发生

这一点很重要。不是你手机上点一下就完事，而是在哪台 Gateway 主机上开工，就在哪台主机上做第一次入住。

gateway host

像真正放床和桌子的那间房。小工人的新家是在这里，不是在外面临时摆拍。

远端 Gateway

如果 Gateway 在别的机器上，那就去那台机器上改 workspace。像搬家后要在新家门口贴名牌。

本地 app 连接远端

app 只是窗口，真正的家具还是放在远端主机那边。

📌 一句话

bootstrapping 的家，是 Gateway 真正住的那台机器，不是随便一台客户端机器。

命令翻译

🛠 这页没有一长串命令，真正的动作是“把第一天的房间搭起来”

官方没把它做成命令百科，而是做成一次性流程。这样更像开箱，不像考试。

🪣

种下基础文件

像先把空房间里的必备便签贴好，免得后面第一次上班时没地方写字。

👂

Q&A 仪式

像小老师慢慢问：“你叫什么、你喜欢怎么被叫、你平时怎么做事？”

🏷

写入身份文件

像把答案写进名字牌、性格卡和用户卡里，之后每次来都能认得你。

🎈

完成后收起封条

仪式做完就收工，不会每次都重新念一遍。

和 onboarding 的关系

🔁 onboarding 先带你进门，bootstrapping 再给你铺床

很多人会把这两个词混在一起。其实它们是前后两步。

onboarding 之后

先把 Gateway、auth、channels、daemon 这些大方向配好，再轮到 bootstrapping 给 agent 自己整理家。

bootstrapping 之前

它还只是刚进门的小工人，还没完全坐稳。

bootstrapping 之后

它才真正知道自己是谁、你是谁、规矩是什么，然后就能更像一个长期值班的助手。

📌 记一句

onboarding 是把房子搭好，bootstrapping 是把住进去的那位小工人叫醒并安排好。

最后总结

🎈 把这页压成一句最短的话

Bootstrapping 就是第一次入住的小仪式：只做一次，跑在 gateway host 上，先摆好身份和家当，再让小工人正式上岗。
