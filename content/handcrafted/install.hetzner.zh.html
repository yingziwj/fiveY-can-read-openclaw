先讲人话版

🇩🇪 Hetzner 这页，其实是在教你“租一台便宜但长期住人的欧洲小机房，把 OpenClaw 装成一台稳定箱子”

它和 GCP 那页在思路上很像：Docker 跑 Gateway，宿主机保存家当，控制台通过 SSH 隧道安全访问。

不同的是，这页更像“简单、实惠、长期住”的传统 VPS 路线。

适合低成本长期 VPS 重点是 root 机、持久目录、Docker 运行箱 查看官方原文

第一站

🏠 这条路的核心就一句：把 Hetzner 当长期住的家，不是随手丢掉的一次性盒子

官方专门写了“不要把它当 disposable infrastructure”，这句很重要。

🔑

root 直连

像你一开始直接拿着总钥匙进屋，先把基础设施搭好。

📦

Docker 箱子

Gateway 住箱子里，但它的家和工作台要放宿主机。

💾

持久目录

像把重要家当存进固定储物间，不让容器换一次就失忆。

🔌

SSH tunnel

默认还是建议走私人电话线，不急着对公网开大门。

第二站

🧱 安装过程其实就是“先让 Docker 住进来，再给 OpenClaw 准备一间不会失忆的家”

核心动作很朴素。

ssh root@YOUR_VPS_IP

像第一次走进你租下的 Hetzner 小机房。

curl -fsSL https://get.docker.com | sh

像先把大箱子工厂装进屋里。

mkdir -p /root/.openclaw/workspace

像先腾出 OpenClaw 的家和工作间。

chown -R 1000:1000 /root/.openclaw

像把这间屋子的使用权交给容器里真正干活的那位住户。

第三站

🗝️ .env 和 compose 文件，合起来就是“门票、门牌、住址和运行规则”

这部分和 GCP 路线是同一家思路。

OPENCLAW_GATEWAY_TOKEN

管理员门票，必须认真生成，别用示例值混过去。

OPENCLAW_CONFIG_DIR

像告诉容器“真正的家在宿主机哪间房”。

OPENCLAW_WORKSPACE_DIR

像工作台位置，换容器也要跟着保留下来。

--allow-unconfigured

先让灯亮起来，不代表正式入住规则已经写完。

第四站

🧑‍💻 访问方式和 IaC 扩展，其实是在讲“你是手工管房，还是再请一个总包公司”

Hetzner 这页还有一个社区 Terraform 方向。

ssh -N -L 18789:127.0.0.1:18789 root@YOUR_VPS_IP

像继续用私人电话线，把控制台拖回本地浏览器。

🧱 Terraform 方案

像再请一个“总包公司”来管地基、云机、防火墙和灾备，让基础设施也能版本化。

📌 官方态度

主文档给的是手工 Docker 主路，Terraform 是社区加成，不是唯一正解。

🎈 收尾一句

Hetzner 这条路主打的不是花哨，而是“便宜、朴素、能长期住”。
