先讲人话版

🎒 Migrating from Claude 这页，是在教你把 Claude 里的书包整理进 OpenClaw

Claude 里可能放着项目说明、个人习惯、MCP 工具、技能和命令。OpenClaw 不会偷偷乱搬，而是先给你看清单，再备份，再真正放进新家。

最重要的提醒是：最好从一个干净的 OpenClaw 开始导入；如果已经有东西，就先看计划，确认冲突，再决定要不要覆盖。

适合 Claude Code / Claude Desktop 用户搬家 重点是预览、备份、冲突、doctor 查看官方原文

第一站

🧾 OpenClaw 会先拿出搬家清单，而不是直接翻你的抽屉

Claude 迁移 provider 的核心动作是：预览每一项、遮住秘密、先做可验证备份。

👀

先预览

--dry-run 像先把要搬的箱子摆出来，让你看清楚会改哪里。

🔒

遮住秘密

MCP 的 env、headers 等敏感字段会在计划和报告里打码。

📦

先备份

真正 apply 前会创建并验证备份，像搬家前先拍照留底。

🛑

有冲突就停

目标位置已经有文件或配置时，默认不会硬塞进去。

第二站

🛤️ 有两条路：向导适合第一次，CLI 适合反复练习

你可以让 onboarding 帮忙，也可以自己用迁移命令一步步控制。

openclaw onboard --flow import

像让搬家向导带路。它发现本地 Claude 状态后，会把 Claude 作为可导入来源摆给你选。

openclaw onboard --import-from claude --import-source ~/.claude

如果 Claude 的东西不在默认位置，就像告诉搬家公司“旧家门牌在这里”。

openclaw migrate claude --dry-run

脚本化场景先跑预览，适合写自动化或想反复确认的人。

openclaw migrate apply claude --yes

看完计划没问题后再应用。这个动作会真的改变 OpenClaw 状态。

第三站

🧩 哪些会搬进新书包，哪些只放进“旧物盒”

这页最适合低龄读者记成两堆：一堆能直接用，一堆只能留给大人慢慢检查。

✅ 会导入：说明和记忆

CLAUDE.md、.claude/CLAUDE.md 会整理进 OpenClaw 的 AGENTS.md，用户级说明会追加到 USER.md。

✅ 会导入：MCP 工具

项目里的 .mcp.json、Claude Code 状态、Claude Desktop 配置里能识别的 MCP server 会被带过来。

✅ 会导入：技能和命令

有 SKILL.md 的 Claude skills 会复制；Claude commands 会变成 OpenClaw skills，但默认不让模型自动调用。

📁 只进档案盒

hooks、宽泛权限、环境默认值、本地私有说明、旧缓存、Desktop 扩展和系统凭据不会自动变成 live 配置。

第四站

🧭 最稳的流程：预览、应用、体检、重启

把官方长步骤压成四个动作，就像搬完家后检查灯、水、门锁。

openclaw migrate claude --dry-run

先看计划：会改什么、跳过什么、哪里有冲突。

openclaw migrate apply claude --yes

确认后再真正搬。遇到已有目标时，除非你明确加 --overwrite，否则它会拒绝硬覆盖。

openclaw doctor

像请医生检查导入后的配置和状态，看看有没有搬歪。

openclaw gateway restart / openclaw status

最后重启 Gateway，看新搬来的说明、MCP servers 和 skills 有没有真的被加载。

最后记住

🎈 Claude 迁移不是“相信旧规则”，而是“把能读懂的东西安全带过来”

OpenClaw 会拒绝自动执行 hooks、相信宽泛权限或破解看不懂的凭据。能搬的先搬，危险的先放进报告，人再决定。
