先讲人话版

Soul format 像给“角色灵魂卡”规定统一纸张

soul 不是一大箱杂物，而是一张主要说明纸：SOUL.md。ClawHub 需要它足够干净，才能读出简介、生成搜索文字，并知道这个 soul 叫什么。

这页很短，但意思很实用：只放该放的文件，名字要像网址那样安全，每次发布都会形成新版本。

适合要发布 soul 的人 重点是 SOUL.md、frontmatter、slug、版本标签 查看官方原文

盒子里放什么

现在只认一张核心说明纸：SOUL.md

官方写得很硬：soul 是单文件格式，文件名可以是 SOUL.md 或 soul.md。额外文件目前会被拒绝，因为系统不想猜“这些附件到底算不算灵魂的一部分”。

SOUL.md

像角色卡正面，写清它是谁、怎么表达、有什么设定。

soul.md

小写名字也可以认。

不要塞附件

现在的规则更像“只收一张纸”，多出来的文件会让发布失败。

纸头小标签

frontmatter 负责给搜索和界面一眼可读的信息

SOUL.md 是 Markdown，可以带 YAML frontmatter。服务器发布时会从这里抽取元数据，其中 description 会变成 UI 和搜索里的简介。

M

Markdown

正文像普通说明书，给人读，也给系统做 embedding。

Y

YAML frontmatter

像贴在纸头上的小标签，让机器快速读懂。

D

description

像一句橱窗介绍，别人搜索和浏览时会先看到它。

名字和大小

slug 要像安全门牌，bundle 也有大小上限

slug 默认从文件夹名来，但必须小写、URL-safe，不能乱放空格和奇怪符号。整体 bundle 上限是 50MB，而用于 embedding 的文字只来自 SOUL.md。

^[a-z0-9][a-z0-9-]*$

像门牌规则：小写字母、数字和短横线，开头也要稳。

50MB

像邮包上限，别把一整柜东西塞进去。

embedding text

只读 SOUL.md，所以关键说明别藏在别处。

版本

每次发布都是一个版本，tag 是贴在版本上的便签

发布会创建新版本，通常用 semver。latest 这类 tag 像可移动便签，指向当前想让大家默认拿到的版本。
