先讲人话版

⚡ MiniMax 这页是在教你二选一：走 OAuth 快速门，还是走 API key 正规门

它不是难在参数多，而是难在入口有两套。官方明显更偏向你先走 Coding Plan 的 OAuth 快速门，因为省得你自己抄 key。

如果你更想稳稳可控，那就走 API key 路线。

适合要接 MiniMax M2.7 的人 重点是 OAuth 入口和 API key 入口的区别 查看官方原文

第一条路

🚀 OAuth 快速门，像“先登录会员卡，再直接进场”

如果你用的是 MiniMax Coding Plan，这条路最省事。

🧩

openclaw plugins enable minimax

像先把 MiniMax 这块接待模块插到系统里。

🔄

openclaw gateway restart

像让值班大厅重新开灯，好让新模块真的上岗。

🪪

openclaw onboard --auth-choice minimax-portal

像正式走会员登录通道，不用你手抄 API key。

🌍

Global / CN

像先选你要去国际馆还是中国馆，两边的门牌号不同。

第二条路

🔑 API key 路线，像“我自己拿钥匙手动配门”

如果你更想把一切写清楚、控得更细，这条路更适合你。

MINIMAX_API_KEY

像你手里的正式钥匙，后面走的是 Anthropic 兼容接口。

baseUrl: https://api.minimax.io/anthropic

像你告诉 OpenClaw：“MiniMax 这扇门在这里，不要敲错楼层。”

api: anthropic-messages

像规定双方说 Anthropic 这套话术，不要切成 OpenAI 那套口音。

MiniMax-M2.7 / -highspeed

一个像标准档，一个像快跑档。都是 M2.7 家族，只是节奏不同。

补一个容易忽略的点

🛟 “fallback 示例”其实是在教你：MiniMax 也可以当备份轮胎

官方举这个例子，是想告诉你 MiniMax 不一定非得当主脑。

主模型

像你平时最常开的那台主力车。

MiniMax fallback

像后备箱里的备胎。主车有事，它顶上。

MiniMax 不只是一扇能进的门，也是一块可以摆在模型阵容里的灵活拼图。

最后总结

🎈 把 MiniMax 这页压成一句最好记的话

想快就走 OAuth 会员门，想稳就走 API key 手动门。接进来后，MiniMax 既能当主脑，也能当备胎。

如果你正在比较“订阅登录”和“传统 API key”两种风格，这页就是最典型的双路线样本。
