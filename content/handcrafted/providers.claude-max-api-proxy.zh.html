先讲人话版

🪞 Claude Max API Proxy，就是给 Claude Max/Pro 订阅套一个“OpenAI 门脸”

它的意思不是让你多买一个模型，而是把你已经有的 Claude 订阅，借一个代理翻译成很多工具都认识的 OpenAI 说话方式。

好处是个人开发时可能更省钱。风险也写得很直白：这条路偏技术兼容，不是最稳的官方生产路线。

适合已经有 Claude Max/Pro 订阅的人 重点是代理、Claude CLI 登录状态、条款风险 查看官方原文

第一步

💸 为什么有人会走这条路？因为它像“包月卡”路线

官方先把账本摊开给你看：按 token 计费像打车按里程收，Claude Max 则更像月卡。

🧾 Anthropic API

更像标准售票口。清楚、正式、适合生产环境，但用得多时账单会跟着涨。

🎟 Claude Max 订阅

更像包月票。个人长期折腾时，可能比按次结账划算。

⚠ 官方的态度

官方没有把这条路说成“稳稳可生产”。它明确提醒你自己看清条款，再决定要不要用。

🎈 先记一句

这页不是在卖新模型，而是在讲“怎么把订阅套成兼容接口”。

第二步

🔁 那张流程图，其实是在说“门口加了一个翻译员”

你的应用本来会说 OpenAI 语言，Claude CLI 会说另一种语言。代理就站在中间做翻译。

你的 App

像顾客，拿着 OpenAI 格式的点单表来敲门。

claude-max-api-proxy

像前台翻译，把点单表改写成 Claude Code CLI 听得懂的话。

Claude Code CLI

像后厨接单员，它会用你已经登录好的订阅身份去真正下单。

回来的结果

代理再把结果翻回 OpenAI 格式，这样外面的工具就不用改嘴型。

第三步

🛠 安装和启动命令，像“先搭前台，再让翻译员上班”

这里的命令不要死背，把它想成开店动作就好。

📦

npm install -g claude-max-api-proxy

像把这位翻译员请到机器里常驻待命。

🪪

claude --version

表面是在看版本，实际上是在确认 Claude CLI 这个人已经在场，而且通常也顺带确认你装好了它。

🚪

claude-max-api

像正式把翻译前台点亮。它会在本地开一个 localhost:3456 的门口。

🩺

curl /health

像敲一下前台玻璃：“有人在吗？”能回就说明门没死。

最后提醒

🎈 把这页压成一句最好记的话

Claude Max API Proxy 是“订阅套 OpenAI 壳”的兼容路线，适合个人折腾省钱，但你要自己承担条款和稳定性判断，不要把它当成默认生产正道。

如果你想走更稳的正式路，就回到 Anthropic、OpenRouter 这类官方 API 入口页去选。
