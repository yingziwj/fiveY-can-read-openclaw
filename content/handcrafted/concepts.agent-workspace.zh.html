先讲人话版

🗂 Agent Workspace 这页，其实是在告诉你：助手的房间里到底该放什么、不该放什么

很多人把 workspace 想成一个普通目录，但这页其实在讲一套生活秩序。哪些文件是房间内的常驻家具，哪些不该堆进来，什么时候该用 Git 做备份。

这页读懂后，你会更知道怎么给 agent 一个长期稳定、可回溯的家。

适合准备长期养一个 agent 的人 重点是房间布局、边界和备份 查看官方原文

第一站

🏠 默认位置和额外文件夹，在讲“主卧在哪、还可以接几个侧房间”

Workspace 有默认住址，也允许接一些额外目录。但扩房间不等于什么都乱塞进来。

默认位置

像主卧。最核心的规则、项目文件和人格线索通常都在这里。

额外文件夹

像侧房或储物间，可以挂更多资料，但还是要知道哪些是常住家具。

Workspace file map

这一段最值钱，因为它不是抽象概念，而是在解释“这个文件到底代表什么角色”。

🎈 一句话

Workspace 不是杂物间，是有门牌和房间分工的住处。

第二站

🚫 What is NOT in the workspace，反而特别重要

很多混乱都来自把不该住在这个房间里的东西硬塞进去。

不是所有状态都该进来

有些认证、运行态、全局配置并不属于 workspace。

边界感的意义

分清房间私人物品和整栋楼公共设施，迁移、备份和排错都会轻很多。

Git backup

官方推荐私有 Git 备份，是因为 workspace 里很多内容值得留历史，但不该随便公开。

🎈 最后记一句

Workspace 是 agent 的家，不是垃圾桶，最好再给这间房配一本私有 Git 相册。

最后总结

🎈 把 Agent Workspace 压成一句最好记的话

该住在里面的要清楚，不该住在里面的更要清楚，房间才会长期稳定。
