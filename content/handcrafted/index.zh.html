先讲人话版

🦞 OpenClaw 首页，其实是在告诉你“这栋楼怎么进、进来后能做什么”

它不是功能目录堆砌，而是一张入口地图。先告诉你去哪儿开始，再告诉你这套系统怎么把聊天、代理和工具串起来。

你可以把它想成一张游乐场门票背面：先看地图，再决定先坐哪个项目。

适合第一次认识 OpenClaw 的人 先看入口，再看工作方式，再看能力清单 查看官方原文

第一站

🚪 先看三个最像“正门”的入口

首页最实用的不是品牌故事，而是这三个卡片：开始安装、跑引导、打开控制台。

1

Get Started

像领门票。先把 OpenClaw 装起来，再把 Gateway 点亮。

2

Run Onboarding

像有个向导牵着你走。`openclaw onboard` 会把配对和通道设置讲清楚。

3

Open the Control UI

像直接走进总控室。聊天、会话、配置都在这里看。

⭐

先记住这个顺序

先装好，再让向导带路，最后再坐到控制台前。

第二站

🧭 这页真正想让你知道的，是 OpenClaw 的三根主梁

它不是单一聊天机器人，而是把消息通道、AI 代理和插件能力拼成一栋楼。

💬 多通道

WhatsApp、Telegram、Discord、iMessage 都只是不同的门。

🧠 Agent-native

它不是“聊完就完”，而是有会话、工具、记忆和多代理路由。

🔒 自托管

楼和钥匙都在你手里，数据不必先交给别人代管。

一句话

OpenClaw 不是聊天软件，它是给聊天门口接上 AI 代理的自建总线。

第三站

🧩 它怎么运作，其实可以想成“门、柜台、后厨”三层

入口页的那张图，核心不是炫技，而是让你知道消息从哪儿来、代理在哪儿工作、控制台在哪儿看结果。

🚪 门

通道负责把消息接进来，比如 Telegram、Discord、WhatsApp。

🖥 柜台

Control UI 负责聊天、会话和配置查看。

🍳 后厨

Gateway 和 agent 在后台决定怎么回、怎么调工具、怎么记住上下文。

最重要的感觉

你看到的是一个聊天窗口，背后其实是整套消息路由和代理工作流。

最后总结

🎈 首页就是一张“从哪里开始”的地图

先安装，后引导，再进控制台。把这条线走通，后面的通道、工具、插件才会真正连成一体。
