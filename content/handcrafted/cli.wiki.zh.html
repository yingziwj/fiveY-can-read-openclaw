先讲人话版

📚 openclaw wiki 像给记忆仓库做一本可查的百科

openclaw wiki 不是普通搜索框。它维护的是 memory-wiki vault：把资料导入、编目录、标来源、找矛盾、查新旧，还能和 Obsidian 接上。

如果 memory 更像每天写下来的原始记录，那么 wiki 就像把这些记录整理成“可查询、可追踪、可复查”的知识库。

适合想维护 memory-wiki vault、查来源、找矛盾、接 Obsidian 的人 重点是 status、doctor、init、ingest、compile、lint、search、get、apply 查看官方原文

第一站

🧭 先记住 4 个日常动作

这页命令很多，但可以先分成四组：检查、放入、整理、查询。

🩺

status / doctor

一个看当前状态，一个做更完整的体检。刚接触时先跑它们。

📥

init / ingest

init 搭好 vault 架子，ingest 把文件或 URL 放进来源层。

🧱

compile / lint

compile 重建索引和摘要，lint 报告矛盾、过期和低置信内容。

🔎

search / get

先搜到候选，再按页面 id 或路径精读具体内容。

第二站

🛠 一组命令看完整工作流

这组命令像从“空书架”到“能查资料”的一条小路线。

常用命令清单
openclaw wiki status
openclaw wiki doctor
openclaw wiki init
openclaw wiki ingest ./notes/alpha.md
openclaw wiki compile
openclaw wiki lint
openclaw wiki search "alpha"
openclaw wiki get entity.alpha --from 1 --lines 80
先检查

status 像看仪表盘，doctor 像打开机盖检查哪里松了。

再导入

ingest 会保留来源信息。以后你看到一句结论，能知道它从哪里来。

再编译

compile 会刷新索引、相关块、dashboard 和给机器看的 digest。

最后查和读

search 负责找候选，get 负责把某一页按行读出来，适合 agent 精确引用。

第三站

✍️ wiki apply 是“别乱改管理区”的安全笔

这页特别提醒：wiki 里有些内容是被管理的，直接手写容易被下次编译覆盖。wiki apply 就是更窄、更安全的修改入口。

写一页 synthesis
openclaw wiki apply synthesis "Alpha Summary" \
  --body "Short synthesis body" \
  --source-id source.alpha
更新 metadata
openclaw wiki apply metadata entity.alpha \
  --source-id source.alpha \
  --status review \
  --question "Still active?"
它适合做什么

创建或更新 synthesis、补 source id、加问题、加矛盾、改 confidence 和 status。

为什么要这样

像只给你一支细笔，让你改该改的字段，而不是把整本书拆开重写。

第四站

🔌 桥接和 Obsidian 是两条外接路线

wiki 不一定孤零零运行。它可以从 Active Memory 的公开产物导入，也可以借 Obsidian CLI 帮你搜和打开笔记。

🌉

wiki bridge import

在 bridge 模式下，把 active memory 插件导出的公开 memory artifacts 拉进 wiki。

📂

unsafe-local import

只从明确允许的本地路径导入。名字里带 unsafe，是提醒你别把范围放太大。

🪨

wiki obsidian ...

可以查状态、搜内容、打开笔记、执行 Obsidian 命令、创建 daily note。

⚙️

配置项

vaultMode、search.backend、bridge.*、obsidian.* 会决定这些路线怎么跑。

最后总结

🎈 把 openclaw wiki 压成一句话

openclaw wiki 是 memory-wiki vault 的维护工具：先检查，再导入，再编译，再 lint，最后用 search 和 get 把有来源的知识读出来。

相关页可以接着看 /plugins/memory-wiki、/concepts/memory 和 /cli/memory。这三页分别讲插件本体、记忆概念和普通 memory CLI。
