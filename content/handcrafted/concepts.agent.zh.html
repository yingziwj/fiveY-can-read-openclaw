先讲人话版

🤖 Agent Runtime 这页，其实是在回答：一个 OpenClaw 助手平时靠什么活着

它不是单纯一段提示词，也不是只会聊天的壳子。一个 agent 有自己的工作区、启动文件、工具箱、技能包、会话本和边界规则。

所以这页像一张“助手解剖图”，告诉你一位 agent 身上到底装了哪些器官。

适合想理解 agent 不是“昵称”的人 重点是工作区、工具、技能和边界 查看官方原文

第一站

📁 Workspace 是助手住的房间，不是随便一个文件夹

官方一开始就强调 workspace required，是在告诉你：agent 进哪个房间，就按哪个房间的规矩说话和做事。

房间里放什么

项目文件、规则文档、技能说明和长期记忆，很多线索都在这里。

为什么不能忽视

因为 agent 不是悬空大脑。它必须站在某个具体房间里。

Bootstrap files

像房间门口贴着的入住手册，进门先看这些，再开始做事。

🎈 一句话

没有 workspace，agent 就像没落地的脑袋。

第二站

🧰 Built-in tools 和 skills，一个是手脚，一个是攻略卡

工具负责真的去做动作，技能负责告诉它遇到某类问题该怎么安排动作顺序。

Built-in tools

像读文件、执行命令、浏览网页这些手脚动作。没有它们，agent 只能嘴上说懂。

Skills

像贴在工具箱盖子上的使用说明：碰到某类问题先看什么、再做什么。

Runtime boundaries

再聪明的 agent 也不是无限自由，它始终活在工具权限、沙箱和审批规则里。

Sessions

像聊天本。它不只是记住说过的话，也影响后面回答的上下文形状。

最后总结

🎈 把 Agent Runtime 压成一句最好记的话

Agent Runtime 讲的不是一个模型参数，而是一个助手活着时整套房间、工具、会话和围栏怎么一起工作。

接着看 /concepts/agent-loop，会更容易理解这些器官是按什么顺序动起来的。
