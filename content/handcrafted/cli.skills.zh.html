先讲人话版

🧰 openclaw skills 像“去技能小卖部翻货、买货、补货、验货”

这页讲的不是抽象的技能系统，而是你站在命令行里，怎么把技能找出来、装进来、更新掉，再确认现在这套技能能不能用。

你可以把它想成一个小卖部。search 是找货，install 是拎回家，update 是换新货，list 和 info 是查你手里到底有什么。

适合想管理本地 skills、从 ClawHub 安装 skills 的人 这页最重要的是分清“找、装、查、更新”四种动作 查看官方原文

第一站

🛒 先看四个最常用动作：找、装、查、更新

官方把最常用的命令放在最前面，就是想让你先学会怎么逛技能货架。

🔍

openclaw skills search "calendar"

像在货架前喊一句：“我要找日历类的技能。”

📦

openclaw skills install <slug>

像把看中的技能盒子直接放进你当前工作区的购物车里。

🔁

openclaw skills update <slug>

像给某个技能盒子换一张新版说明书。

📋

openclaw skills list

像翻翻自己家里已经摆好的技能盒子。

第二站

🧺 这个小卖部里，货架动作很简单

官方给的命令列表不长，但每一个都非常像现实动作。

search

像先逛一圈，看看有没有你想要的技能名。

install

像把货架上的盒子抱回当前项目桌上。

update

像确认手里的技能盒子是不是有新版本了，有的话就替换。

list / list --eligible

像翻家里现有的技能清单。--eligible 则是只看那些“现在条件满足、可以上场”的盒子。

第三站

🏷 info、check，像把盒子拿起来看标签、摇一摇

安装之前看标签，安装之后摇一摇，都是为了避免拿回去一盒坏的。

🪪

openclaw skills info <name>

像把技能盒子拿到眼前，看看名字、说明和里面写了什么。

🧪

openclaw skills check

像把盒子轻轻晃一下，确认它现在能不能顺利上场。

📚

本地 vs 远处

官方强调 list/info/check 看的还是本地、当前工作区能见到的技能，不是只看云端货架。

⭐

先记一句

info 是看说明，check 是看能不能用。

第四站

🧰 这些命令背后的现场动作，其实都很直白

这页官方只给了一排命令，因为它希望你把技能当成“可以管理的实体盒子”，而不是模糊概念。

openclaw skills search "calendar"

像在小卖部里问：“有没有日历类的小盒子？”

openclaw skills install <slug>

像把找到的盒子搬回当前工作区，让 OpenClaw 以后能直接看到。

openclaw skills update --all

像把架子上所有旧说明书一起换新。

openclaw skills list --eligible

像只看今天已经满足条件、可以上手的技能盒子。

第五站

🏠 search / install / update 都是在说“ClawHub 货架和你桌上的盒子怎么来回搬”

官方特别点明了这三者直接和 ClawHub 打交道，所以它们像一条搬运线。

🛒 search

先在货架前找货，不要一上来就把整排货架搬回家。

📥 install

像把选中的技能放进当前工作区的技能目录。

🔁 update

像去确认这盒技能是不是已经出新版本，出的话就换上。

📌 这页真正想说的

skills 这页是让你管理“本地可用的技能盒子”，不是只看一个远端目录。

第六站

🧿 如果你想把它记得更牢，就把它想成四个小动作

找、装、查、验，这四个词基本就能把这一页的动作串起来。

找

search

去货架前问一圈。

装

install

把盒子搬回桌上。

查

list / info

看自己有什么、盒子写了什么。

验

check / update

看看能不能用、要不要换新版。

最后总结

🎈 把 openclaw skills 压成一句话

openclaw skills 就是 OpenClaw 的技能小卖部：先找货，再把技能盒子搬回当前工作区，然后定期检查和更新。

如果你下一页只想继续看一页，我建议看 /tools/skills。那一页讲的是 skill 在系统里到底是什么，而这页讲的是怎么把它管理起来。
