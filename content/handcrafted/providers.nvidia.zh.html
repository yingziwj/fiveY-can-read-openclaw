先讲人话版

🟢 NVIDIA 这页很像“拿 NGC 门卡去开 OpenAI 兼容大门”

它的接法并不花。重点就是：准备好 NVIDIA_API_KEY，再把默认模型切成 NVIDIA 那边的型号。

你可以把它想成“门长得像 OpenAI，但里面站着的是 NVIDIA 的模型”。

适合想接 Nemotron / NeMo 模型的人 重点是 NGC key、base URL、默认模型切换 查看官方原文

第一步

🔑 先准备 NGC 门卡，再让 OpenClaw 知道你要走 NVIDIA 这扇门

官方推荐先用环境变量，不推荐把 token 直接塞进命令参数里到处留痕。

📦

export NVIDIA_API_KEY="nvapi-..."

像先把门卡放进口袋，而不是举在街上让每个人都看见。

🚪

openclaw onboard --auth-choice skip

这里更像“跳过前台问答，后面自己直接设模型”。

🧠

openclaw models set ...

像把默认大脑正式换成 NVIDIA 那位老师。

⚠

别把 token 硬塞进命令里

官方专门提醒，是因为那样容易留在 shell 历史和进程列表里。

第二步

🏠 配置片段，其实只是把门牌号和默认老师写清楚

配置看着长，翻成人话就是“这扇门在哪、说哪种话、默认请谁上场”。

https://integrate.api.nvidia.com/v1

像 NVIDIA 这栋楼的门牌号。

api: openai-completions

像告诉 OpenClaw：“进去后，按 OpenAI 这套说话方式交流。”

nvidia/nvidia/llama-3.1-nemotron-70b-instruct

这串长名字就是完整路线图：先到 NVIDIA 商场，再找到具体柜台。

自动启用

官方说得很省心：只要这把 key 在，provider 可以自动亮起来。

第三步

📚 那几条模型 ID，像“今天大厅里常见的几位老师”

先不用全记，先知道默认老师是谁就够了。

默认款

nvidia/llama-3.1-nemotron-70b-instruct 像大厅正中间那位被默认安排上场的老师。

可替换款

meta/llama-3.3-70b-instruct、nvidia/mistral-nemo-minitron-8b-8k-instruct 像同一栋楼里的别的柜台。

🎈 先别被列表吓到

先跑通一位老师，再慢慢换，不要一上来把整栋楼都背下来。

最后总结

🎈 把 NVIDIA 这页压成一句最好记的话

这页最核心的动作就是：把 NGC 门卡放稳，指定 NVIDIA 模型，然后把 OpenClaw 带去那扇 OpenAI 兼容门口。

如果你也在用本地模型机房，那下一页更该对比 Ollama 和 SGLang，不要把远程大楼和本地机房混一起看。
