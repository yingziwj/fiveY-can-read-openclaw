先讲人话版

🔄 Agent Loop 这页，其实是在画“助手每接到一条任务，脑子里会走几圈”

这不是数学循环课，而是 OpenClaw 的工作节奏图。消息从哪进来、排队怎么排、提示词怎么拼、钩子什么时候插手，都是在这里串起来的。

如果 Agent Runtime 是身体结构，Agent Loop 就是身体实际怎么动起来。

适合想理解一次任务从进入到完成全过程的人 重点是队列、提示词组装和钩子 查看官方原文

第一站

🚪 Entry points 很多，但最后都会汇进同一条工作流水线

CLI、聊天通道、Web 界面、自动任务，表面入口不同，进入 loop 以后还是要按统一流程排队和处理。

不同入口

像不同门口进楼的人，有的从大门，有的从侧门，但最后都要经过前台登记。

统一处理

进入 loop 以后，系统开始按标准节奏走，不会因为入口不同就完全换脑回路。

How it works

这一节就是整页的总导航图，告诉你接下来会依次经过哪些检查站。

🎈 一句话

入口可以很多，主循环只有一套。

第二站

📬 Queueing + concurrency 在讲“先排队，再决定能不能多开几张工位”

这段很关键，因为它决定系统是稳稳地一条条处理，还是允许多件事并发推进。

排队

像窗口前先拿号。先后顺序、会不会打架、会不会把同一个会话挤乱，都靠这里控制。

并发

像多开几张工位一起干，但不是越多越好。开太多会把上下文和顺序搅乱。

Prompt assembly

像开工前把任务单、规则卡、上下文纸条和工具说明摊在桌上，再交给模型老师开始答。

🎈 最后记一句

Agent Loop 决定消息进来以后，系统到底按什么节奏组织思考、工具和状态。

最后总结

🎈 把 Agent Loop 压成一句最好记的话

它讲的不是抽象循环，而是一次任务从进门、排队、拼提示词到被 hooks 观察和拦截的真实流水线。
