先讲人话版

🧾 Kilo Gateway 像一个模型总前台：一把钥匙，后面很多柜台

你不用一家家模型厂商分别办卡，而是先在 Kilo 这里领一张总门卡。它再根据任务，把请求送去合适的模型。

所以它最有趣的地方，不是“又多了一个 provider”，而是“它会替你挑后厨”。

适合想少管模型路由的人 重点是 KILOCODE_API_KEY 和 kilo/auto 查看官方原文

第一步

🔑 先领一把 Kilo 总门卡

官方给你的动作非常直：去 Kilo 后台拿 key，然后交给 OpenClaw。

🛠

openclaw onboard --kilocode-api-key <key>

像把门卡直接递给新手向导。

📦

KILOCODE_API_KEY

像总前台通行证。它不是某个单独模型的小钥匙。

🏠

推荐放环境变量

像把门卡放进值班室抽屉，Gateway 自己上班时也能摸到。

🎈

这一页最先记住什么

不是先选模型，而是先领总门卡。

第二步

🤖 kilocode/kilo/auto 为什么最特别？因为它像“自动分单员”

官方默认不是随便给的，它代表一种自动路由思路。

kilocode/kilo/auto

像总前台里的调度员，收到任务后会自己判断该往哪位老师那里送。

规划 / 调试 / 编排

更像会被送去更擅长深度思考的那位老师。

写代码 / 浏览 / 常规任务

更像会被送去更适合日常出手、速度更稳的那位老师。

🎈 最值得懂的点

你不是在选“一个死模型”，而是在选“一个会自动转单的模型入口”。

第三步

🏬 那串长模型名，像“商场 / 店铺 / 柜台”路线图

看到 kilocode/anthropic/claude-sonnet-4 这种长名字时，不要慌，它只是在写清楚你最后会被送到哪。

kilocode/anthropic/claude-sonnet-4

第一段是商场，第二段是哪家店，第三段才是具体柜台。

/models kilocode

像问前台：“你们家今天都上了哪些老师？”

Bearer token

底层就是每次请求都拿着那张 Kilo 门卡去验票。

最后总结

🎈 把 Kilo Gateway 这页压成一句最好记的话

Kilo Gateway 像模型总前台。你交一把总门卡进去，既可以直接点具体柜台，也可以偷懒用 kilocode/kilo/auto 让它自动分单。

如果你更喜欢自己精确挑模型，而不是让总前台帮你转单，就去看普通 provider 的单店页面。
