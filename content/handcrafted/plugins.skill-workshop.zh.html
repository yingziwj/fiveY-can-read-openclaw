先讲人话版

Skill Workshop，像一个会把“这次学到的方法”整理成小手册的工作台

这页讲的是实验性插件。它会观察一次工作中值得复用的流程、修正和坑点，帮你把它们整理成 workspace 里的 SKILL.md。

它不是记住“你喜欢什么”的记忆系统，而是把“下次遇到类似任务该怎么做”写成步骤手册。

实验功能，默认关闭 建议先用 pending 模式审一遍再写入 查看官方原文

第一站

先分清 Memory、Skill 和 Skill Workshop

Memory

像小本子，记事实、偏好、人物和过去发生的事。

Skill

像操作手册，写“遇到这种任务时按这些步骤做”。

Skill Workshop

像手册编辑台，把一次有价值的经验变成可复用的 skill 草稿。

为什么默认关闭

因为自动写文件有风险。先审，再应用，才适合多数工作区。

第二站

提案生命周期像“草稿箱到正式书架”

1

Capture

从对话和文件变化里发现可能值得记录的流程。

2

Review

启用 LLM reviewer 或人工查看，判断它是不是真的有用。

3

Pending

先放进待处理区，像草稿还没上架。

4

Apply

确认没问题后写入 SKILL.md 或支持文件。

第三站

工具命令是一组“手册管理员按钮”

status

看工作台现在开没开、会不会自动应用。

list_pending / inspect

看看有哪些草稿，点开读清楚再决定。

suggest

主动请工作台根据当前经验提出一份 skill 草稿。

apply / reject

通过就放进书架，不合适就退回或丢弃。

第四站

什么时候不要自动应用

不信任的工作区

如果仓库来源不确定，不要让插件自动写入长期规则。

高风险流程

涉及凭据、部署、删除、发布的经验，必须人工看清楚。

一次性修补

只适合这一次的小绕路，不值得沉淀成未来规则。

一句话收住

Skill Workshop 很像聪明的笔记员，但笔记要不要放进正式手册，仍然要有人把关。
